Epic 01: 用户体系与授权

US-01-01

个人用户注册

P0
作为个人用户，我想要使用手机号注册账号，以便后续兑换卡密并使用系统。
验收标准 (Acceptance Criteria)
  • 提供手机号、密码、姓名完成注册。
  • 密码最少 8 位，须包含字母和数字。
  • 同一手机号不能重复注册。
  • 注册成功后进入卡密兑换页。
US-01-02

用户登录与会话管理

P0
作为用户，我想要使用手机号和密码登录，以便访问我的授权内容。
验收标准 (Acceptance Criteria)
  • 手机号 + 密码登录成功后签发 Token，有效期 7 天（后端可配置）。
  • 同一账号仅允许一个活跃会话，新设备登录时旧设备会话立即失效。
  • 旧设备下次请求时返回登录失效提示。
边缘场景 (Edge Cases)
  • 连续登录失败 3 次后账号锁定 5 分钟，锁定期满自动解除。
  • 锁定不影响已有授权和数据。锁定只阻止新登录，不影响已有活跃会话。
US-01-03

企业员工账号创建（单个）

P0
作为运营管理员，我想要在运营后台创建企业员工账号，以便员工可以使用企业授权。
验收标准 (Acceptance Criteria)
  • 输入手机号、姓名、初始密码，绑定企业组织节点。
  • 同一时间一个员工只能绑定一个企业。
  • 手机号已存在且未绑定企业时，将现有账号绑定当前企业。
  • 手机号已绑定其他企业时，拒绝创建并提示。
US-01-04

企业员工批量导入

P1
作为运营管理员，我想要通过 Excel 批量导入员工账号，以便快速完成大量员工入库。
验收标准 (Acceptance Criteria)
  • 支持 .xlsx 上传。必填：手机号、姓名。可选：初始密码。
  • 未填密码时系统生成默认密码（手机号后 6 位 + 固定后缀）。
  • 单次上限 500 条。
  • 按行逐条处理：不存在→新建；已存在未绑定→绑定；已绑定其他企业→跳过；格式错误→跳过。
  • 导入完成后显示结果摘要（新建 N / 绑定 N / 跳过 N + 原因）。
  • 导入记录审计日志。
US-01-05

密码重置

P1
作为运营管理员，我想要在后台重置用户密码，以便帮助忘记密码的用户恢复登录。
验收标准 (Acceptance Criteria)
  • 运营后台可选择用户并重置密码。
  • 重置后不强制首次登录修改密码。
  • 重置操作记录审计日志。
业务规则 (Business Rules)
  • 用户自助修改密码、找回密码首期不做。
US-01-06

停用/启用用户账号

P0
作为运营管理员，我想要停用/启用学员账号，以便管控用户登录权限。
验收标准 (Acceptance Criteria)
  • 停用后用户不能登录。
  • 停用时该账号所有活跃会话立即失效。
  • 数据、授权、作答记录、报告、AI 日志保留。
  • 可重新启用。
  • 停用/启用记录审计日志。
边缘场景 (Edge Cases)
  • 停用时用户正在模拟考试中，系统自动保存作答并终止，记录状态为 `terminated`。
  • 用户刷新时提示"账号已被停用，请联系管理员"。
US-01-07

卡密兑换

P0
作为个人用户，我想要输入卡密兑换授权，以便获得对应专业/等级的内容访问权。
验收标准 (Acceptance Criteria)
  • 输入卡密后兑换成功，授权从当天开始计算。
  • 兑换不区分大小写，自动去除前后空格。
  • 兑换成功后卡密状态变为已使用，不可撤销。
  • 错误提示区分：卡密不存在 / 已使用 / 已过兑换截止日。
边缘场景 (Edge Cases)
  • 允许同一用户兑换多张同专业/等级卡密，多条授权独立存在，有效期取并集（最晚到期日为准）。
  • 企业员工也可兑换个人卡密，获得个人授权。个人授权和企业授权并存。
  • 卡密兑换在数据库层做原子操作，防止并发兑换同一张卡密。
US-01-08

卡密批量生成

P0
作为运营管理员，我想要批量生成卡密，以便分发给个人用户。
验收标准 (Acceptance Criteria)
  • 同批次指定专业、等级、授权时长、可选兑换截止日。
  • 一次最多 100 张。
  • 卡密 8 位大写字母数字，避免易混淆字符。
  • 后台列表可查看明文卡密，支持搜索和按状态筛选。
  • 卡密状态：`unused` / `used` / `expired`（未使用且兑换截止日已过）。截止日判定以服务器时间（UTC+8）为准。
US-01-09

卡密购买引导页

P1
作为无授权用户，我想要在登录后看到卡密兑换页和购买联系方式，以便获取卡密。
验收标准 (Acceptance Criteria)
  • 卡密输入框 + 兑换按钮 + 已兑换授权列表。
  • 显示运营配置的联系电话、微信号、二维码、文案。
  • 购买联系方式由运营后台配置，修改记录审计日志。
US-01-10

企业组织树管理

P0
作为运营管理员，我想要维护企业组织树，以便按真实组织结构管理企业客户。
验收标准 (Acceptance Criteria)
  • 支持新增、编辑、停用/启用企业节点。
  • 组织树最多 4 级（省-地市-县区-基层）。
  • 禁止循环引用，保存时校验。
  • 停用时可选择是否级联下级。
边缘场景 (Edge Cases)
  • 企业停用时，如有员工正在考试/练习，系统自动保存并终止，状态为 `terminated`。即使员工同时拥有个人卡密授权覆盖同一专业/等级，基于企业授权的考试/练习仍立即终止。终止后员工可基于个人授权重新开始。
US-01-11

企业授权管理

P0
作为运营管理员，我想要创建和管理企业授权记录，以便控制企业员工的内容访问权。
验收标准 (Acceptance Criteria)
  • 指定购买主体、授权范围（本企业及下级 / 指定企业列表）、专业、等级、账号额度、有效期。
  • 不允许同专业/等级的授权范围重叠到同一企业节点。
  • 授权状态：有效 / 已过期 / 已取消。
  • 支持取消企业授权。取消后受影响员工立即失去访问权。
  • 额度校验在数据库层做原子操作，防止并发超额。
依赖项 (Dependencies)
  • US-01-10
US-01-12

员工调动与离职

P1
作为运营管理员，我想要变更员工企业归属或解绑，以便处理员工调动和离职。
验收标准 (Acceptance Criteria)
  • 员工企业可变更，历史学习记录保持作答时企业快照。
  • 解绑后不再继承企业授权，不默认停用账号。
  • 有个人卡密授权的用户解绑后仍可作为个人用户登录。
  • 额度释放和占用在同一事务中原子执行，防止并发超额。
US-01-13

授权访问与展示

P0
作为学员，我想要看到所有有效授权对应的内容，以便进行学习和练习。
验收标准 (Acceptance Criteria)
  • 个人授权和企业授权取并集，任一有效即可访问。
  • 我的授权列表分别显示来源、专业/等级、有效期、状态。
  • 授权失效后前端隐藏对应内容，后台数据保留。
  • 重新获得授权后恢复可见。
边缘场景 (Edge Cases)
  • 授权失效时（自然到期、主动取消、企业停用），如用户正在对应专业/等级的练习或模拟考试中，立即终止（详见 US-03-04 EC-1 和 US-03-05 EC-2）。
US-01-14

授权到期站内提示

P2
作为学员，我想要在授权即将到期时收到站内提示，以便及时续费。
验收标准 (Acceptance Criteria)
  • 授权到期前 15 天开始提示。
  • 首页顶部通知条展示。
  • 同一条授权每天最多展示一次，关闭后当天不再重复。
  • 个人授权引导兑换新卡密，企业授权引导联系企业/客服。

Epic 02: 题库、试卷与内容管理

US-02-01

题目新建与编辑

P0
作为内容老师，我想要通过结构化表单创建和编辑题目，以便建设题库。
验收标准 (Acceptance Criteria)
  • 按题型引导表单，包含完整基础字段（题干、选项、答案、解析、评分点等共 16 项，见 02 §2.5）。
  • 富文本编辑题干、解析、标准答案，支持插入图片（≤5MB，JPG/PNG/GIF/WebP）和基础表格。
  • 题干/解析上限 10000 字符，超出时编辑器提示。
  • 可关联材料、知识点、标签。
  • 未被已发布试卷引用的题目可编辑；已锁定题目不能编辑只能复制新题。
  • 判断题内部存储为正确/错误，学员端显示为 A.正确 / B.错误。后台录入 A/B 时系统自动转换。
  • 主观题评分点含"描述"和"分值"两个字段。评分点分值允许 0.5 粒度。首期不做关键词、可接受表述、扣分规则。
US-02-02

题目停用与复制

P1
作为内容老师，我想要停用不再使用的题目或复制已锁定题目，以便管控题库质量。
验收标准 (Acceptance Criteria)
  • 停用后不能被新试卷选择，已发布试卷中的题目不受影响。
  • 已锁定题目可停用。
  • 可复制题目为新题（独立编辑）。
US-02-03

题库搜索与筛选

P0
作为内容老师，我想要按多维度搜索筛选题目，以便快速定位和组卷。
验收标准 (Acceptance Criteria)
  • 支持按专业、等级、科目、题型、状态筛选。
  • 支持按关键字搜索题干文本。
  • 支持按标签和知识点筛选。
  • 组卷选题时默认按当前试卷专业/等级/科目预筛选。
US-02-04

多选题评分规则配置

P1
作为内容老师，我想要为多选题配置评分规则，以便灵活控制评分策略。
验收标准 (Acceptance Criteria)
  • 支持"全对得分"和"漏选部分得分，错选不得分"两种规则。
  • 默认为"全对得分"。
US-02-05

填空题评分方式配置

P1
作为内容老师，我想要选择填空题的评分方式，以便区分自动匹配和 AI 评分。
验收标准 (Acceptance Criteria)
  • 支持"自动匹配"和"AI 评分"两种方式。
  • 切换时保留已录入答案，不自动清除。
  • 切换为 AI 评分时需补充评分点；切换为自动匹配时需确认标准答案。
  • 报告中标明评分方式。
  • 填空题评分方式是母题固有属性，题目加入试卷后不允许在试卷中切换。如需变更，回题库修改母题或复制新题。
US-02-06

材料库管理

P0
作为内容老师，我想要独立维护材料库，以便材料可跨题目和试卷复用。
验收标准 (Acceptance Criteria)
  • 材料含标题、正文（富文本，上限 30000 字符）、表格、图片、专业/等级/科目、状态。
  • 材料仅在被已发布试卷引用后才锁定。锁定后需复制新材料编辑。
  • 可停用旧材料。停用后已发布试卷中引用该材料的题目不受影响（使用快照），学员端正常展示。
  • 可查看材料被引用的题目和试卷列表。
US-02-07

草稿试卷组卷

P0
作为内容老师，我想要从题库选题组卷，以便构建完整试卷。
验收标准 (Acceptance Criteria)
  • 题目加入草稿时复制当时快照（含关联材料）。
  • 草稿内可调整题目顺序、设置分值、调整大题/题组归属。
  • 不允许编辑题干、选项、客观题答案和解析。
  • 主观题评分点可按试卷调整（增删改），不反向修改母题。
  • 大题分值由子题自动累加。
依赖项 (Dependencies)
  • US-02-01, US-02-06
US-02-08

试卷发布与强校验

P0
作为内容老师，我想要发布试卷时系统自动校验完整性，以便确保试卷质量。
验收标准 (Acceptance Criteria)
  • 每道题必须设置分值。
  • 总分 = 所有计分子题分值合计。
  • 主观题评分点分值合计 = 该题满分。
  • 多空填空每空分值合计 = 题目总分。
  • 至少 1 道计分题目。不额外限制最少主观题/客观题数量，也不设置最少总分限制。
  • 不允许空大题/模块。
  • 校验不通过不能发布。
  • 发布后内容锁定，被引用的母题锁定。
US-02-09

试卷下架与删除

P0
作为内容老师，我想要下架或删除不再使用的试卷，以便管控内容。
验收标准 (Acceptance Criteria)
  • 下架后学员不能新开始练习/考试。
  • 未完成的练习/考试立即终止。
  • 已完成的历史报告保留可查看。
  • 草稿且无引用可删除；已发布/有记录的试卷不可删除。
US-02-10

复制试卷

P1
作为内容老师，我想要复制已发布/已下架试卷为新草稿，以便快速创建相似试卷。
验收标准 (Acceptance Criteria)
  • 复制时题干、选项、客观题答案和解析从母题重新取当前版本快照；主观题评分点从源试卷中复制（保留源试卷中调整过的评分点描述和分值），而非从母题重新取默认评分点。
  • 名称追加"（副本）"后缀，状态为草稿。
  • 源试卷中已停用母题仍纳入，标记提示"该题目已停用"。
US-02-11

试卷元数据与原始文件

P1
作为内容老师，我想要维护试卷元数据并绑定原始文件，以便追溯和管理。
验收标准 (Acceptance Criteria)
  • 基础字段（名称、专业、等级、科目、总分）必填，时长可选（10~300 分钟）。
  • 扩展元数据（类型、年份、来源等）可选。
  • 可绑定多个原始文件（试卷/材料/答案/解析等），标记用途。
  • 学员端不展示原始文件。

Epic 03: 学员端答题与体验

US-03-01

学员首页

P0
作为学员，我想要登录后看到授权范围内的试卷列表，以便选择学习内容。
验收标准 (Acceptance Criteria)
  • 展示有效授权范围内的专业/等级。
  • 多个授权时记住上次选择，首次引导选择。
  • 当前专业/等级下按科目（理论/技能）分组显示试卷。
  • 每套试卷提供"练习"和"模拟考试"两个入口。
  • 试卷列表按发布时间倒序。
  • 无授权用户直接进入卡密兑换页。
US-03-02

理论卷练习

P0
作为学员，我想要逐题练习理论卷并获得即时反馈，以便巩固知识。
验收标准 (Acceptance Criteria)
  • 一题一页，答完即显示对错、正确答案、解析。
  • 答错自动进入错题本 + 自动生成 AI 讲解。
  • 答对时显示答案/解析 + "需要讲解"按钮。
  • 客观题不允许再答一次。
依赖项 (Dependencies)
  • US-04-04（AI 讲解）
US-03-03

技能卷练习

P0
作为学员，我想要按材料题组练习技能卷，以便提升实操能力。
验收标准 (Acceptance Criteria)
  • 按材料题组/大题一组一页，材料置顶可折叠/展开。
  • 主观题提交后可选"AI 提示并重答一次"或"直接查看评分"。
  • 最多 1 次重答，第二次后给出最终评分。
  • 主观题不进入错题本。
依赖项 (Dependencies)
  • US-04-01（AI 评分），US-04-04（AI 提示）
US-03-04

练习暂停与继续

P1
作为学员，我想要离开后再继续上次练习进度，以便灵活安排学习时间。
验收标准 (Acceptance Criteria)
  • 离开后已保存进度保留 15 天。
  • 再次进入同一试卷练习时提示"继续/重新开始"。
  • 超过 15 天进度自动清除。
  • 同一试卷同一用户只能有一个活跃练习进度。多设备场景下，后打开的设备加载服务端已有进度。
边缘场景 (Edge Cases)
  • 练习中授权到期时立即终止，系统自动保存已完成的作答到后台（用于 AI 日志追溯），学员端不可查看。提示“授权已失效”。
US-03-05

模拟考试答题

P0
作为学员，我想要进行整卷模拟考试，以便模拟真实考试场景。
验收标准 (Acceptance Criteria)
  • 作答中不显示对错、答案、解析。

P0

验收标准 (Acceptance Criteria)

  • 同一次作答只触发一次评分，成功结果固定不重复调用。

  • 未作答主观题直接 0 分，不调用 AI。
业务规则 (Business Rules)

  • 首期最多重试 3 次。超限后标记"评分失败（已达重试上限）"。
  • 部分/全部失败时报告状态为 `scoring_partial_failed`。
  • 全部失败时客观题部分可展示但标记"主观题部分未评分"。

US-06-13

后台角色与权限

P0
作为超级管理员，我想要管理后台用户和角色，以便控制后台访问权限。
验收标准 (Acceptance Criteria)
  • 三种角色：超级管理员（全局）、运营管理员（运营后台）、内容老师（内容后台）。
  • 一个用户可拥有多个角色。
  • 内容老师首期不做专业/等级权限隔离。
  • 后台用户由超级管理员创建，不使用学员注册流程。
  • 可新建、编辑角色、停用后台用户。
  • 后台用户会话安全：Token 有效期 8 小时；允许同一账号多设备登录；连续登录失败 5 次锁定 15 分钟，锁定只阻止新登录，不影响已有活跃会话。
  • 后台账号与学员账号是两套独立体系。同一自然人如需同时使用学员端和后台端，需分别持有两个账号。
业务规则 (Business Rules)
  • 首个超级管理员通过系统初始化脚本或种子数据创建，不通过后台界面。
