## 1. 主观题 AI 评分

### US-04-01: 主观题 AI 评分（模拟考试）

**作为**学员，**我想要**交卷后系统自动评分主观题，**以便**获得完整考试成绩。

**验收标准**：

- [ ] AC-1: 整卷交卷后统一异步评分所有主观题。
- [ ] AC-2: 评分结果保存：总分、每个评分点（是否命中 + 实得分 + AI 原因）、总评语、改进建议、引用来源、模型信息、Prompt 版本。
- [ ] AC-3: 同一次作答只触发一次评分，成功结果固定不重复调用。
- [ ] AC-4: 界面标识"AI 自动评分"。
- [ ] AC-5: 未作答主观题直接 0 分，不调用 AI。

**业务规则**：

- BR-1: 首期不支持人工改分、申诉、复核。
- BR-2: 评分点部分得分粒度 0.5 分，总分不超题目满分。AI 返回不符合 0.5 倍数的，四舍五入到最近的 0.5；后端校验确保总分不超满分。

**优先级**：P0

---

### US-04-02: 评分失败与重试

**作为**学员，**我想要**在评分失败后重试，**以便**获得完整报告。

**验收标准**：

- [ ] AC-1: 失败后标记评分失败，用户可稍后点击重新评分。
- [ ] AC-2: 重试只针对失败的提交，不对已成功结果重评。
- [ ] AC-3: 首期最多重试 3 次。超限后标记"评分失败（已达重试上限）"。
- [ ] AC-4: 部分/全部失败时报告状态为 `scoring_partial_failed`。
- [ ] AC-5: 全部失败时客观题部分可展示但标记"主观题部分未评分"。

**边缘场景**：

- EC-1: 单题超时 60 秒视为评分失败。
- EC-2: 评分任务使用异步队列，按 FIFO 处理。

**优先级**：P0

---

### US-04-03: 模拟考试评分中页面

**作为**学员，**我想要**交卷后看到评分进度，**以便**知道何时可以查看报告。

**验收标准**：

- [ ] AC-1: 进入"评分中"页面，不先展示客观题半成品分数。
- [ ] AC-2: 提供"刷新结果"按钮（首期不自动轮询）。
- [ ] AC-3: 评分完成→进入报告；仍在进行→继续等待；失败→显示失败数量 + "重试评分"。
- [ ] AC-4: 可离开页面，稍后从记录列表进入。
- [ ] AC-5: 评分中页面提示“评分可能需要几分钟，请耐心等待或稍后查看”。

**优先级**：P0

## 2. AI 讲解与提示

### US-04-04: 客观题 AI 讲解

**作为**学员，**我想要**在练习中获得 AI 讲解，**以便**理解题目考点。

**验收标准**：

- [ ] AC-1: 答错后自动生成 AI 讲解。
- [ ] AC-2: 答对后提供"需要讲解"按钮手动触发。
- [ ] AC-3: 讲解基于题目 + 答案 + 解析 + RAG 命中资料。
- [ ] AC-4: 展示参考来源（资料名称 + 章节路径）。
- [ ] AC-5: RAG 命中不足时显示依据不足提示。
- [ ] AC-6: 首期不做缓存，每次触发均调用 AI。

**边缘场景**：

- EC-1: 主模型和备用模型均失败时，提示“AI 讲解暂时不可用，请稍后重试”。用户可手动重试或跳过继续答下一题。
- EC-2: AI 讲解失败不阻塞答题流程。失败记录进入 AI 调用日志。

**优先级**：P0

---

### US-04-05: 主观题 AI 提示与重答

**作为**学员，**我想要**在主观题练习中获得改进提示并重答，**以便**提升答题质量。

**验收标准**：

- [ ] AC-1: 第一次提交→AI 评分→显示结果。
- [ ] AC-2: 可选"AI 提示"→获得改进方向（不直接给答案）。
- [ ] AC-3: 可重新作答（最多一次）→AI 评分→显示最终结果。
- [ ] AC-4: 两次作答和评分均保存，报告显示最终（第二次）结果。

**优先级**：P0

## 3. AI 知识点推荐

### US-04-06: AI 知识点推荐

**作为**内容老师，**我想要**保存题目时系统自动推荐知识点，**以便**高效建立题目与知识体系的关联。

**验收标准**：

- [ ] AC-1: 首次保存时异步触发，或手动点击"AI 推荐"。
- [ ] AC-2: 推荐 1~5 个知识点，标记置信度（高/中/低）。
- [ ] AC-3: 展示为候选列表，老师可勾选确认/修改/忽略。
- [ ] AC-4: 未确认的推荐仍参与报告分析，标注"AI 推荐，待确认"。
- [ ] AC-5: 发布试卷时不强制逐题确认知识点。
- [ ] AC-6: 推荐失败不阻塞题目保存/发布。

**边缘场景**：

- EC-1: AI 知识点推荐为异步任务，不阻塞题目保存。
- EC-2: 推荐过程中题目被再次编辑保存，以最新一次保存触发的推荐为准，旧推荐结果丢弃。
- EC-3: 推荐过程中题目被停用，推荐结果丢弃，不写入。

**优先级**：P1

## 4. 模型管理

### US-04-07: 模型供应商配置

**作为**超级管理员，**我想要**配置 AI 模型供应商，**以便**灵活切换模型。

**验收标准**：

- [ ] AC-1: 配置模型供应商、名称、API Key、启用状态、功能用途映射。
- [ ] AC-2: API Key 密文展示（只显示后 4 位）。
- [ ] AC-3: 配置变更记录审计日志。
- [ ] AC-4: 只允许超级管理员操作。

**业务规则**：

- BR-1: 客观题讲解/提示可配置 fallback 备用模型。
- BR-2: 主观题评分不自动 fallback，避免评分口径变化。
- BR-3: 模型切换不影响已开始的评分任务（启动时锁定配置）。

**优先级**：P0

---

### US-04-08: Prompt 模板管理

**作为**开发团队，**我想要**通过配置文件管理 Prompt 模板，**以便**独立迭代 Prompt。

**验收标准**：

- [ ] AC-1: 评分、讲解、提示、知识点推荐分别独立 Prompt 模板。
- [ ] AC-2: 每个模板有版本号。
- [ ] AC-3: 每次 AI 调用记录 Prompt 版本。
- [ ] AC-4: 首期不做后台编辑界面。

**优先级**：P1
