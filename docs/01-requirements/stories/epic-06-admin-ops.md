## 1. 管理后台基础

### US-06-01: 后台通用交互

**作为**后台用户，**我想要**一致的列表、筛选和操作体验，**以便**高效工作。

**验收标准**：

- [ ] AC-1: 列表默认分页 20 条，支持 50 / 100。
- [ ] AC-2: 表头点击切换升序/降序。
- [ ] AC-3: 筛选条件变化后自动刷新。
- [ ] AC-4: 批量操作需二次确认弹窗。
- [ ] AC-5: 危险操作红色按钮 + 二次确认。
- [ ] AC-6: 操作成功/失败 Toast 提示。
- [ ] AC-7: 关键写操作（授权创建/调整、员工批量导入、卡密生成）使用乐观锁或数据库原子操作防止并发冲突。冲突时提示“数据已被其他操作更新，请刷新后重试”。

**优先级**：P0

## 2. 运营后台

### US-06-02: 用户管理

**作为**运营管理员，**我想要**管理用户账号，**以便**维护用户数据。

**验收标准**：

- [ ] AC-1: 列表显示手机号、姓名、注册时间、状态、绑定企业、授权状态。
- [ ] AC-2: 支持按手机号/姓名搜索，按状态和用户类型筛选。
- [ ] AC-3: 可查看用户详情（基本信息 + 授权列表 + 企业绑定）。
- [ ] AC-4: 可停用/启用账号、重置密码。
- [ ] AC-5: 首期不做用户删除。

**依赖**：US-01-05, US-01-06

**优先级**：P0

---

### US-06-03: 企业组织管理界面

**作为**运营管理员，**我想要**可视化管理企业组织树，**以便**直观维护组织结构。

**验收标准**：

- [ ] AC-1: 树形结构展示企业组织。
- [ ] AC-2: 支持新增、编辑、停用/启用节点（停用可选级联）。
- [ ] AC-3: 企业节点下查看/管理员工列表（搜索、筛选、创建、批量导入、解绑）。
- [ ] AC-4: 查看企业节点的授权情况。

**依赖**：US-01-10

**优先级**：P0

## 4. 日志

**作为**运营管理员，**我想要**查看后台操作审计日志，**以便**追溯关键操作。

**验收标准**：

- [ ] AC-1: 列表支持按操作类型、操作人、时间范围筛选。
- [ ] AC-2: 支持按关键字搜索操作对象。

- [ ] AC-5: 首期不设日志保留期限，数据永久保留。

**作为**运营管理员，**我想要**查看 AI 调用日志和成本汇总，**以便**监控 AI 使用和成本。

**验收标准**：

## 5. 权限

### US-06-13: 后台角色与权限

**作为**超级管理员，**我想要**管理后台用户和角色，**以便**控制后台访问权限。

**验收标准**：

- [ ] AC-1: 三种角色：超级管理员（全局）、运营管理员（运营后台）、内容老师（内容后台）。
- [ ] AC-2: 一个用户可拥有多个角色。
- [ ] AC-3: 内容老师首期不做专业/等级权限隔离。
- [ ] AC-4: 后台用户由超级管理员创建，不使用学员注册流程。
- [ ] AC-5: 可新建、编辑角色、停用后台用户。
- [ ] AC-6: 后台用户会话安全：Token 有效期 8 小时；允许同一账号多设备登录；连续登录失败 5 次锁定 15 分钟，锁定只阻止新登录，不影响已有活跃会话。
- [ ] AC-7: 后台账号与学员账号是两套独立体系。同一自然人如需同时使用学员端和后台端，需分别持有两个账号。

**业务规则**：

- BR-1: 首个超级管理员通过系统初始化脚本或种子数据创建，不通过后台界面。

**优先级**：P0
