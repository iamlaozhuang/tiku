# Epic 01: 用户体系与授权

> 源文档：[01-user-auth.md](../modules/01-user-auth.md)

## 1. 统一账号

### US-01-01: 个人用户注册

**作为**个人用户，**我想要**使用手机号注册账号，**以便**后续兑换卡密并使用系统。

**验收标准**：
- [ ] AC-1: 提供手机号、密码、姓名完成注册。
- [ ] AC-2: 密码最少 8 位，须包含字母和数字。
- [ ] AC-3: 同一手机号不能重复注册。
- [ ] AC-4: 注册成功后进入卡密兑换页。

**优先级**：P0

---

### US-01-02: 用户登录与会话管理

**作为**用户，**我想要**使用手机号和密码登录，**以便**访问我的授权内容。

**验收标准**：
- [ ] AC-1: 手机号 + 密码登录成功后签发 Token，有效期 7 天（后端可配置）。
- [ ] AC-2: 同一账号仅允许一个活跃会话，新设备登录时旧设备会话立即失效。
- [ ] AC-3: 旧设备下次请求时返回登录失效提示。

**边缘场景**：
- EC-1: 连续登录失败 3 次后账号锁定 5 分钟，锁定期满自动解除。
- EC-2: 锁定不影响已有授权和数据。锁定只阻止新登录，不影响已有活跃会话。

**优先级**：P0

---

### US-01-03: 企业员工账号创建（单个）

**作为**运营管理员，**我想要**在运营后台创建企业员工账号，**以便**员工可以使用企业授权。

**验收标准**：
- [ ] AC-1: 输入手机号、姓名、初始密码，绑定企业组织节点。
- [ ] AC-2: 同一时间一个员工只能绑定一个企业。
- [ ] AC-3: 手机号已存在且未绑定企业时，将现有账号绑定当前企业。
- [ ] AC-4: 手机号已绑定其他企业时，拒绝创建并提示。

**优先级**：P0

---

### US-01-04: 企业员工批量导入

**作为**运营管理员，**我想要**通过 Excel 批量导入员工账号，**以便**快速完成大量员工入库。

**验收标准**：
- [ ] AC-1: 支持 .xlsx 上传。必填：手机号、姓名。可选：初始密码。
- [ ] AC-2: 未填密码时系统生成默认密码（手机号后 6 位 + 固定后缀）。
- [ ] AC-3: 单次上限 500 条。
- [ ] AC-4: 按行逐条处理：不存在→新建；已存在未绑定→绑定；已绑定其他企业→跳过；格式错误→跳过。
- [ ] AC-5: 导入完成后显示结果摘要（新建 N / 绑定 N / 跳过 N + 原因）。
- [ ] AC-6: 导入记录审计日志。

**优先级**：P1

---

### US-01-05: 密码重置

**作为**运营管理员，**我想要**在后台重置用户密码，**以便**帮助忘记密码的用户恢复登录。

**验收标准**：
- [ ] AC-1: 运营后台可选择用户并重置密码。
- [ ] AC-2: 重置后不强制首次登录修改密码。
- [ ] AC-3: 重置操作记录审计日志。

**业务规则**：
- BR-1: 用户自助修改密码、找回密码首期不做。

**优先级**：P1

## 2. 账号停用

### US-01-06: 停用/启用用户账号

**作为**运营管理员，**我想要**停用/启用学员账号，**以便**管控用户登录权限。

**验收标准**：
- [ ] AC-1: 停用后用户不能登录。
- [ ] AC-2: 停用时该账号所有活跃会话立即失效。
- [ ] AC-3: 数据、授权、作答记录、报告、AI 日志保留。
- [ ] AC-4: 可重新启用。
- [ ] AC-5: 停用/启用记录审计日志。

**边缘场景**：
- EC-1: 停用时用户正在模拟考试中，系统自动保存作答并终止，记录状态为 `terminated`。
- EC-2: 用户刷新时提示"账号已被停用，请联系管理员"。

**优先级**：P0

## 3. 个人卡密授权

### US-01-07: 卡密兑换

**作为**个人用户，**我想要**输入卡密兑换授权，**以便**获得对应专业/等级的内容访问权。

**验收标准**：
- [ ] AC-1: 输入卡密后兑换成功，授权从当天开始计算。
- [ ] AC-2: 兑换不区分大小写，自动去除前后空格。
- [ ] AC-3: 兑换成功后卡密状态变为已使用，不可撤销。
- [ ] AC-4: 错误提示区分：卡密不存在 / 已使用 / 已过兑换截止日。

**边缘场景**：
- EC-1: 允许同一用户兑换多张同专业/等级卡密，多条授权独立存在，有效期取并集（最晚到期日为准）。
- EC-2: 企业员工也可兑换个人卡密，获得个人授权。个人授权和企业授权并存。
- EC-3: 卡密兑换在数据库层做原子操作，防止并发兑换同一张卡密。

**优先级**：P0

---

### US-01-08: 卡密批量生成

**作为**运营管理员，**我想要**批量生成卡密，**以便**分发给个人用户。

**验收标准**：
- [ ] AC-1: 同批次指定专业、等级、授权时长、可选兑换截止日。
- [ ] AC-2: 一次最多 100 张。
- [ ] AC-3: 卡密 8 位大写字母数字，避免易混淆字符。
- [ ] AC-4: 后台列表可查看明文卡密，支持搜索和按状态筛选。
- [ ] AC-5: 卡密状态：`unused` / `used` / `expired`（未使用且兑换截止日已过）。截止日判定以服务器时间（UTC+8）为准。

**优先级**：P0

---

### US-01-09: 卡密购买引导页

**作为**无授权用户，**我想要**在登录后看到卡密兑换页和购买联系方式，**以便**获取卡密。

**验收标准**：
- [ ] AC-1: 卡密输入框 + 兑换按钮 + 已兑换授权列表。
- [ ] AC-2: 显示运营配置的联系电话、微信号、二维码、文案。
- [ ] AC-3: 购买联系方式由运营后台配置，修改记录审计日志。

**优先级**：P1

---

## 4. 企业组织与授权

### US-01-10: 企业组织树管理

**作为**运营管理员，**我想要**维护企业组织树，**以便**按真实组织结构管理企业客户。

**验收标准**：
- [ ] AC-1: 支持新增、编辑、停用/启用企业节点。
- [ ] AC-2: 组织树最多 4 级（省-地市-县区-基层）。
- [ ] AC-3: 禁止循环引用，保存时校验。
- [ ] AC-4: 停用时可选择是否级联下级。

**边缘场景**：
- EC-1: 企业停用时，如有员工正在考试/练习，系统自动保存并终止，状态为 `terminated`。即使员工同时拥有个人卡密授权覆盖同一专业/等级，基于企业授权的考试/练习仍立即终止。终止后员工可基于个人授权重新开始。

**优先级**：P0

---

### US-01-11: 企业授权管理

**作为**运营管理员，**我想要**创建和管理企业授权记录，**以便**控制企业员工的内容访问权。

**验收标准**：
- [ ] AC-1: 指定购买主体、授权范围（本企业及下级 / 指定企业列表）、专业、等级、账号额度、有效期。
- [ ] AC-2: 不允许同专业/等级的授权范围重叠到同一企业节点。
- [ ] AC-3: 授权状态：有效 / 已过期 / 已取消。
- [ ] AC-4: 支持取消企业授权。取消后受影响员工立即失去访问权。
- [ ] AC-5: 额度校验在数据库层做原子操作，防止并发超额。

**依赖**：US-01-10

**优先级**：P0

---

### US-01-12: 员工调动与离职

**作为**运营管理员，**我想要**变更员工企业归属或解绑，**以便**处理员工调动和离职。

**验收标准**：
- [ ] AC-1: 员工企业可变更，历史学习记录保持作答时企业快照。
- [ ] AC-2: 解绑后不再继承企业授权，不默认停用账号。
- [ ] AC-3: 有个人卡密授权的用户解绑后仍可作为个人用户登录。
- [ ] AC-4: 额度释放和占用在同一事务中原子执行，防止并发超额。

**优先级**：P1

## 5. 授权访问统一规则

### US-01-13: 授权访问与展示

**作为**学员，**我想要**看到所有有效授权对应的内容，**以便**进行学习和练习。

**验收标准**：
- [ ] AC-1: 个人授权和企业授权取并集，任一有效即可访问。
- [ ] AC-2: 我的授权列表分别显示来源、专业/等级、有效期、状态。
- [ ] AC-3: 授权失效后前端隐藏对应内容，后台数据保留。
- [ ] AC-4: 重新获得授权后恢复可见。

**边缘场景**：
- EC-1: 授权失效时（自然到期、主动取消、企业停用），如用户正在对应专业/等级的练习或模拟考试中，立即终止（详见 US-03-04 EC-1 和 US-03-05 EC-2）。

**优先级**：P0

---

### US-01-14: 授权到期站内提示

**作为**学员，**我想要**在授权即将到期时收到站内提示，**以便**及时续费。

**验收标准**：
- [ ] AC-1: 授权到期前 15 天开始提示。
- [ ] AC-2: 首页顶部通知条展示。
- [ ] AC-3: 同一条授权每天最多展示一次，关闭后当天不再重复。
- [ ] AC-4: 个人授权引导兑换新卡密，企业授权引导联系企业/客服。

**优先级**：P2
