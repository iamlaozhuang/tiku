# Epic 05: RAG 与知识库

> 源文档：[05-rag-knowledge.md](../modules/05-rag-knowledge.md)

## 1. RAG 检索

### US-05-01: 混合检索与 rerank

**作为**系统，**我想要**通过混合检索为 AI 提供相关资料片段，**以便**AI 评分和讲解有据可依。

**验收标准**：

- [ ] AC-1: 检索前按专业、等级、资源状态过滤。
- [ ] AC-2: 关键词 + 向量语义检索初召回，合并去重后 rerank。
- [ ] AC-3: 取 Top 3 片段给 AI。
- [ ] AC-4: rerank 不可用时降级到融合排序，日志标记。
- [ ] AC-5: 按等级相关性排序：精确匹配 > 多等级通用 > 等级为空。

**优先级**：P0

---

### US-05-02: RAG 命中状态与权限过滤

**作为**系统，**我想要**标记命中状态并过滤权限，**以便**保证引用质量和数据安全。

**验收标准**：

- [ ] AC-1: 返回 `evidence_status`：`sufficient` / `weak` / `none`。
- [ ] AC-2: 命中不足时 AI 不伪造引用，显示依据不足提示。
- [ ] AC-3: 学员端只检索当前有效授权范围内的资料。
- [ ] AC-4: 授权失效后不允许调用对应内容 AI。

**优先级**：P0

---

## 2. 资源管理

### US-05-03: 资源上传与转换

**作为**运营管理员，**我想要**上传教材文件并转换为 Markdown，**以便**建设 AI 知识库。

**验收标准**：

- [ ] AC-1: 支持 DOCX / Markdown / PPTX / 可抽取文本 PDF。
- [ ] AC-2: 单文件上限 50MB。
- [ ] AC-3: 转换失败时记录原因，运营可查看。
- [ ] AC-4: 转换失败的资源不允许发布、不参与 RAG。
- [ ] AC-5: 资源必须绑定专业（必填），等级可选。
- [ ] AC-6: PPTX 转换效果依赖原始文件质量，建议优先上传 DOCX 或 Markdown 格式。PPTX 仅作为兼容选项。

**优先级**：P0

---

### US-05-04: Markdown 校对与发布

**作为**运营管理员，**我想要**在线校对 Markdown 并发布，**以便**确保知识库内容质量。

**验收标准**：

- [ ] AC-1: 查看/编辑 Markdown 源文（标题、正文、列表、表格）。
- [ ] AC-2: 调整章节层级。
- [ ] AC-3: 保存草稿 + 人工确认发布。
- [ ] AC-4: 未发布资源不参与 RAG。
- [ ] AC-5: 发布后不自动重建向量，状态为"已发布，待建向量"。

**优先级**：P0

---

### US-05-05: 资源状态机

**作为**系统，**我想要**按状态机管理资源生命周期，**以便**保证流程严谨。

**验收标准**：

- [ ] AC-1: 9 种状态：`uploaded` → `converting` → `draft` / `conversion_failed` → `published` → `indexing` → `rag_ready` / `index_failed` → `disabled`。
- [ ] AC-2: 只允许合法状态转换（见 05 §6.6）。
- [ ] AC-3: 支持停用/启用，停用后不参与 RAG，启用后如向量存在立即恢复。

**优先级**：P0

---

### US-05-06: 向量重建

**作为**运营管理员，**我想要**手动触发向量重建，**以便**更新 RAG 检索内容。

**验收标准**：

- [ ] AC-1: 异步切分 chunk → 写入向量库 → 状态变为 `rag_ready`。
- [ ] AC-2: 重建期间 RAG 继续使用旧向量。
- [ ] AC-3: 首次构建期间该资源不参与检索。
- [ ] AC-4: 重建成功后原子切换到新向量。
- [ ] AC-5: 失败时继续使用旧向量，显示失败原因。
- [ ] AC-6: 停用后 chunk 不删除，恢复时快速可用。恢复时系统检查 chunk 是否仍存在，如已不存在（如向量库重建或清理），资源状态恢复到“已发布，待建向量”，需手动触发重建。
- [ ] AC-7: 内容已更新但向量未重建时，AI 引用该资源的结果增加标记“引用内容可能非最新版本”。

**优先级**：P0

---

### US-05-07: Chunk 切分

**作为**系统，**我想要**自动按 Markdown 结构切分 chunk，**以便**生成高质量检索片段。

**验收标准**：

- [ ] AC-1: 按标题层级保留章节路径。
- [ ] AC-2: 过长段落自动拆分，过短段落可合并。
- [ ] AC-3: 每个 chunk 保存：资料名称、章节路径、专业、等级、资源 ID、chunk 序号。
- [ ] AC-4: 切分参数为系统配置项，运营不可调整。

**优先级**：P0

## 3. 知识点体系

### US-05-08: 知识点树管理

**作为**内容老师，**我想要**维护知识点树，**以便**建立题目和知识体系的关联。

**验收标准**：

- [ ] AC-1: 按专业维护，节点可标记适用等级。
- [ ] AC-2: 支持新增、编辑、排序、停用。不允许删除。
- [ ] AC-3: 支持移动（修改父节点）和重命名，操作记录审计日志。
- [ ] AC-4: 最多 5 级，建议单专业不超 2000 个。
- [ ] AC-5: 停用后不推荐给新题，已绑定题目不受影响。
- [ ] AC-6: 历史报告使用快照，不受移动/停用影响。
- [ ] AC-7: 如某专业尚未维护知识点树（空树），AI 知识点推荐返回空结果，不报错。知识点移动后，AI 推荐基于当前最新路径检索。

**优先级**：P0

---

### US-05-09: 知识点薄弱分析

**作为**学员，**我想要**在模拟考试报告中看到知识点薄弱分析，**以便**针对性学习。

**验收标准**：

- [ ] AC-1: 按知识点统计正确率/得分率。
- [ ] AC-2: AI 推荐的知识点也参与统计（标注"仅供参考"）。
- [ ] AC-3: 无知识点题目仍纳入总分和题型统计。
- [ ] AC-4: 历史报告不重算，使用快照。

**依赖**：US-04-06, US-03-07

**优先级**：P1
