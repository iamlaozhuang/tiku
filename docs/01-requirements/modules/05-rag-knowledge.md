## 1. RAG 首期要求

首期 RAG 必须上线，且检索命中有要求。

RAG 用于：

- AI 讲解。
- AI 提示。
- 主观题评分依据。
- AI 知识点推荐。

学员端 AI 引用来源展示粒度：

- 资料名称 + 章节路径。

示例：

```text
参考来源：
《烟草专卖管理师（三至五级）》 > 第三篇 三级 > 第一章 证件管理 > 第一节 许可证办理
```

## 2. 混合检索与 rerank

首期 RAG 检索采用混合检索：

- 关键词检索。
- 向量语义检索。
- rerank。

流程：

1. 检索前按专业、等级、资源状态过滤。
2. 关键词检索 + 向量检索初召回。
3. 合并候选、去重。
4. rerank 对候选片段重新排序。
5. 取最终 Top 3 片段给 AI。

Top-K：

- 首期固定 Top 3。
- AI 讲解、提示、知识点推荐、评分都使用 Top 3。
- 学员端最多展示这 3 个引用来源。

降级：

- 如果 rerank 服务不可用，可降级到融合排序。
- 日志标记"rerank 未使用"。

## 3. RAG 命中状态

RAG 返回 `evidence_status`：

- `sufficient`：命中充分，正常展示引用。
- `weak`：命中弱，展示依据不足提示。
- `none`：无命中，不展示引用，只展示依据不足提示。

命中不足时：

- AI 不能伪造引用。
- 主观题评分可以继续，但标记"未检索到充分教材依据"。
- AI 讲解/提示可以继续，但显示"以下内容基于题目与标准答案生成，未检索到明确教材依据"。
- AI 知识点推荐可以写入，但标记低置信度。

## 4. RAG 权限过滤

学员端：

- 只能检索用户当前有效授权范围内的专业/等级资料。
- 如果用户没有该题所属专业/等级授权，不能通过 AI 间接获取资料。
- 授权失效后，不允许继续调用对应内容 AI。

内容老师录题：

- 按题目选择的专业/等级检索。
- 老师首期不做专业权限限制。
- 不提供跨专业手动选择检索范围。

## 5. RAG 质量验收

第一版采用人工抽样评估，后续再建设测试集。

首期验收标准采用宽松标准：

- 多数样例能命中相关资料即可。
- 抽样覆盖专卖、营销。
- AI 讲解、提示、知识点推荐、主观评分都抽样检查。
- 多数样例需命中正确专业/等级下的相关资料。
- 引用展示为资料名称 + 章节路径。
- 命中不足时必须显示依据不足，不能伪造引用。
- 不要求每个问题都命中预期章节。

后续测试集：

- 标准问题。
- 期望章节。
- 最低命中阈值。
- 期望引用来源。

## 6. 资源与 Markdown 知识库

### 6.1 资源定位

教材、课件、知识点文档首期不面向学员开放学习，只作为后台资源和 AI 知识库使用。

学员端不提供：

- 教材 PDF 阅读。
- 课程章节。
- 课件播放。
- 学习进度。

### 6.2 文件访问

所有资料文件不公开访问，必须鉴权。

包括：

- 原始 PDF / DOCX / PPTX / Markdown。
- Markdown 派生内容。
- RAG chunk。

规则：

- 对象存储使用私有桶。
- 后台下载资源文件必须登录并有权限。
- 下载通过后端生成临时签名 URL 或后端流式转发。
- AI 和 RAG 服务内部鉴权。
- 日志中避免暴露长期可访问文件 URL。
- 学员端只能看到 AI 引用的资料名称和章节路径，不直接访问教材文件。

### 6.3 知识库支持格式

首期进入知识库的资源格式：

- DOCX。
- Markdown。
- PPTX。
- 可抽取文本 PDF。

PPTX 转换说明：

- PPTX 转换效果依赖原始文件质量。以文字内容为主的 PPT 效果较好，以图表、动画、复杂排版为主的 PPT 可能丢失大量内容。
- 建议内容管理员优先上传 DOCX 或 Markdown 格式。PPTX 仅作为兼容选项。

扫描型 PDF：

- 首期不支持 OCR。
- 解析失败时标记为"解析失败/需处理"。
- 内容管理员或运营支持人员线下处理原始文件后重新上传可转换版本。

转换失败处理：

- 转换失败时记录失败原因（如文件损坏、格式不支持、内容为扫描图片等）。
- 内容管理员可查看失败原因。
- 内容管理员可删除失败资源并重新上传处理后的文件。
- 转换失败的资源不允许发布、不参与 RAG。

文件上传限制：

- 单个资源文件上传大小上限 50MB。
- 超出上限时提示“文件过大，请压缩或拆分后上传”。

### 6.4 资源元数据

资源进入知识库前必须绑定专业，等级可选。

规则：

- 专业必填。
- 等级可多选，也可为空表示该专业通用资料。
- 例如《烟草专卖管理师（三至五级）》可绑定三级、四级、五级。
- 资源需标记资料类型：
  - 教材。
  - 课件。
  - 知识点文档。
  - 讲稿。
  - 其他。

RAG 检索：

- 先按专业过滤。
- 优先检索匹配等级资料。
- 等级为空或覆盖多等级的资料可作为该专业通用资料参与检索。
- 学员无某等级授权时，不能检索只属于该等级的资料。
- 检索时按等级相关性排序：精确匹配等级的 chunk > 多等级通用 chunk > 等级为空的通用 chunk。rerank 阶段可将等级匹配度作为排序因子之一。

### 6.5 Markdown 标准中间格式

教材、课件、知识点文档进入 RAG 前，统一转换为 Markdown。

保留两份内容：

- 原始文件：用于留档、下载、版权追溯。
- Markdown：用于章节识别、知识点抽取、RAG 切分、引用来源、人工校对。

规则：

- Markdown 必须由系统从上传文件转换生成。
- 不允许内容管理员直接新建/粘贴无来源 Markdown 资源。
- 如果资料本身是 Markdown，也必须作为文件上传，再读取生成 Markdown 草稿。
- 文件转换失败不能入库，不允许手动上传替代 Markdown。
- 内容管理员需要线下处理原始文件后重新上传。

### 6.6 Markdown 校对与发布

内容后台提供 Markdown 编辑/校对界面。

流程：

```text
上传原始文件
→ 转换 Markdown 草稿
→ AI/规则抽取章节树
→ 内容管理员在线校对 Markdown 和章节层级
→ 人工点击发布
→ 手动触发重建向量
→ RAG 可用
```

能力：

- 查看 Markdown 源文。
- 编辑标题、正文、列表、表格文本。
- 调整章节层级。
- 保存草稿。
- 人工确认发布。
- 记录更新时间和操作人。

不做：

- 多人协同。
- 版本 diff。
- 复杂富文本排版。
- Markdown 版本管理。

发布规则：

- 转换成功后必须人工确认发布，才能进入知识库。
- 未发布资源不参与 RAG 检索。
- 发布后不自动重建向量。
- 发布后状态为"已发布，待建向量"。
- 内容管理员手动点击“重建向量”。

资源状态流转：

- 已上传（`uploaded`）：原始文件已上传，等待转换。
- 转换中（`converting`）：正在转 Markdown。
- 转换失败（`conversion_failed`）：转换出错，需重新上传或线下处理。
- 草稿（`draft`）：Markdown 已生成，待人工校对。
- 已发布（`published`）：人工确认发布，待建向量。
- 向量构建中（`indexing`）：正在切分和写入向量库。
- RAG 可用（`rag_ready`）：向量构建成功，可正常检索。
- 向量构建失败（`index_failed`）：构建出错，RAG 使用旧向量（如有）或不可用。
- 已停用（`disabled`）：内容管理员手动停用，不参与 RAG 检索。

合法转换：uploaded → converting → draft / conversion_failed；draft → published → indexing → rag_ready / index_failed；rag_ready → published（重新编辑发布后回到待建向量）；任何状态 → disabled（停用）；disabled → 恢复到停用前状态。

### 6.7 向量重建

知识库发布后，由内容管理员手动点击"重建向量"。

重建流程：

- 系统异步切分 chunk。
- 写入向量库和关键词索引。
- 构建成功后状态变为"RAG 可用"。
- 构建失败显示失败状态和原因。

如果已 RAG 可用的资源重新编辑发布，但还没重建向量：

- RAG 继续使用旧向量。
- 状态显示"内容已更新，向量待重建"。
- 新向量构建成功后切换到新向量。
- 重建失败则继续使用旧向量。

引用标记：

- 状态为“内容已更新，向量待重建”时，AI 引用该资源的来源增加标记“引用内容可能非最新版本”。
- 内容后台资源列表高亮提示有待重建向量的资源数量。

向量重建期间行为：

- 重建期间，RAG 继续使用旧向量（如有）正常检索。
- 首次构建（资源无旧向量）期间，该资源不参与 RAG 检索。
- 重建成功后原子切换到新向量。

资源停用后向量处理：

- 资源停用后，其 chunk 不参与新的 RAG 检索（检索时按资源状态过滤）。
- 停用不立即删除向量库中的 chunk，以便恢复时快速可用。
- 重新启用后，如向量仍存在，立即恢复可检索。
- 重新启用资源时，系统检查该资源的 chunk 是否仍存在于向量库。
- 如 chunk 已不存在（如向量库重建或清理），资源状态恢复到“已发布，待建向量”，需内容后台具备资源写权限的 `content_admin` 或 `super_admin` 手动触发重建。

### 6.8 Chunk 切分

首期自动按 Markdown 标题和段落切分，不提供人工调整。

规则：

- 优先按标题层级保留章节路径。
- 段落过长时自动拆分。
- 段落过短时可合并相邻内容。
- 每个 chunk 保存：
  - 资料名称。
  - 章节路径。
  - 专业。
  - 等级。
  - 资源 ID。
  - chunk 序号。

切分参数（如 chunk 目标长度、最大长度、最小长度、相邻 chunk 重叠策略）作为系统配置项，由技术团队按效果优化，运营不可调整。

## 7. 知识点体系

### 7.1 知识点来源

每个专业的教材、考试大纲、知识点文档统一上传到系统。

AI 可从资料中抽取知识点树草稿，但正式知识点由平台运营确认发布。

首期策略：

- 公共知识点库由内容后台统一维护。
- 内容老师/出题人可在内容后台维护公共知识库；系统运营后台不作为首期资源/知识库主入口。
- 不支持机构/老师私有知识点。
- 老师录题时使用公共知识点库。
- AI 自动推荐题目关联知识点，老师可事后修改。

AI 知识点推荐规则：

- 推荐时机：题目首次保存时自动触发，或老师手动点击“AI 推荐”。
- 推荐数量：每道题推荐 1~5 个知识点。
- 推荐结果展示为候选列表，老师可勾选确认、修改或忽略。
- 推荐基于题干 + 参考答案 + RAG 检索结果。
- 推荐结果标记置信度（高/中/低）。
- 老师未确认的推荐知识点仍参与报告分析，但标注“AI 推荐，待确认”。

### 7.2 知识点树结构

知识点树按专业维护，节点可标记适用等级。

原因：

- 许多教材覆盖三至五级，不适合拆成完全独立的等级树。
- 不同等级之间会有重复知识点。
- 专业一棵树更利于知识体系完整维护。
- 等级标签表达适用范围。

首期不做等级继承逻辑。

规则：

- 每个专业维护当前知识点树。
- 节点字段建议：
  - 名称。
  - 父节点。
  - 排序。
  - 适用等级。
  - 来源资源。
  - 状态。
- 题目绑定知识点时，按题目专业筛选知识点，等级只作为辅助过滤/标记。
- 知识点树层级首期建议最多 5 级（专业-篇-章-节-知识点）。
- 单个专业下知识点总数首期不做硬限制，但建议不超过 2000 个。
- 如某专业尚未维护知识点树（空树），AI 知识点推荐返回空结果，不报错。题目知识点绑定为可选字段，允许为空。

### 7.3 知识点节点管理

首期知识点树不做版本管理。

知识点节点：

- 可新增。
- 可编辑。
- 可排序。
- 可停用。
- 不允许删除。

停用后：

- 不再推荐给新题。
- 不在录题选择器中默认显示。
- 已绑定该知识点的题目不受影响。
- 历史报告保存知识点名称/路径快照，不受停用影响。

知识点移动与重命名：

- 知识点节点可移动（修改父节点）和重命名。
- 已绑定该知识点的题目自动关联到新路径。
- 历史报告中的知识点路径使用报告快照，不受移动/重命名影响。
- 移动和重命名操作记录审计日志。
- 知识点移动后，AI 推荐基于当前最新路径检索，不使用旧路径。

### 7.4 知识点分析

模拟考试报告中的知识点薄弱分析可以基于 AI 自动绑定知识点生成，并标注"仅供参考"。

规则：

- 题目有知识点绑定时，按知识点统计正确率/得分率。
- 知识点来自 AI 推荐也可以参与统计。
- 无知识点题目仍纳入总分、题型、大题模块统计。
- 后续老师修正知识点后，新报告按新绑定计算。
- 历史报告不重算，使用报告快照。
