## 1. 统一账号

所有学员账号都使用手机号作为唯一登录标识。

个人用户注册字段：

- 手机号
- 密码
- 姓名

企业员工账号：

- 由平台运营创建或批量导入。
- 使用手机号 + 初始密码登录。
- 绑定企业组织节点。
- 运营单个创建员工时手动输入初始密码。
- 首期一个员工同一时间只能绑定一个企业。

规则：

- 同一手机号不能重复创建账号。
- 个人用户后续加入企业时，可以在原账号上绑定企业，不需要新账号。
- 批量导入员工：
  - 支持 Excel（.xlsx）上传。必填字段：手机号、姓名。可选字段：初始密码（未填则系统生成默认密码，如手机号后 6 位 + 固定后缀）。
  - 单次导入上限 500 条。
  - 导入按行逐条处理：手机号不存在则新建账号并绑定企业；手机号已存在且未绑定企业则绑定（不覆盖原有密码和学习记录）；手机号已绑定其他企业则跳过并提示；手机号格式错误则跳过。
  - 导入完成后显示结果摘要：新建 N 个、绑定 N 个、跳过 N 个（附跳过原因）。
  - 导入记录审计日志。
- 企业员工也可以兑换个人卡密，在同一账号下获得个人授权。
- 手机号首期不可修改。
- 用户自助修改密码、找回密码首期不做。
- 忘记密码由运营后台重置。
- 重置密码后不强制首次登录修改密码。

密码规则：

- 首期密码最少 8 位，须包含字母和数字。
- 不做复杂度强制（如大小写混合、特殊字符）。
- 后续可升级密码策略。

登录安全：

- 连续登录失败 3 次后，账号锁定 5 分钟。
- 锁定期间该手机号不能登录。
- 锁定不影响已有授权和数据。
- 锁定期满后自动解除。
- 锁定只阻止新登录，不影响已有活跃会话。已登录用户可继续使用。

会话管理：

- 登录成功后签发 Token，有效期由后端配置，首期建议 7 天。
- 同一账号仅允许一个活跃会话。新设备登录时，旧设备会话立即失效，旧设备下次请求时返回登录失效提示。

## 2. 账号停用

账号支持后台停用：

- 超级管理员/运营管理员可停用学员账号。
- 停用后用户不能登录。
- 数据、授权、作答记录、报告、AI 日志保留。
- 可重新启用。
- 停用/启用记录审计日志。
- 停用账号时，该账号所有活跃会话立即失效。如果用户正在模拟考试中，系统自动保存当前作答并终止考试会话，记录状态为“已终止”（`terminated`）。
- 用户刷新或下次请求时提示“账号已被停用，请联系管理员”。

授权过期和账号停用是两套机制：

- 授权控制内容访问。
- 账号停用控制登录。

## 3. 个人卡密授权

### 3.1 卡密模式

个人用户首期采用卡密兑换，不做在线支付。后续在线支付可以复用同一套授权模型：支付成功后创建个人授权记录。

规则：

- 个人用户自助注册后，没有授权时直接进入卡密兑换页。
- 卡密只用于兑换授权，不作为登录凭证。
- 一个账号可以兑换多个卡密，获得多个专业/等级授权。
- 允许同一用户兑换多张同专业/等级卡密。多条个人授权独立存在，有效期取并集（即以最晚到期日为准），不自动接续。
- 卡密兑换成功后不可撤销、不可换绑。
- 兑换错专业/等级时，系统不修改原授权，售后通过线下补发新卡或人工赠送新授权处理。

### 3.2 卡密权益

卡密绑定粒度：

- 专业
- 等级
- 授权时长
- 可选兑换截止日期

示例：

- 专卖三级一年
- 营销三级半年

授权有效期：

- 从兑换成功当天开始计算。
- 例如用户在 2026-05-12 兑换一年卡，到期日为 2027-05-12。

兑换截止日期：

- 生成卡密时可选。
- 未设置截止日：卡密未使用前长期可兑换。
- 设置截止日：超过截止日不能兑换。截止日判定以服务器时间（UTC+8）为准。
- 不论是否设置截止日，授权期都从兑换日开始算。

### 3.3 卡密生成与展示

首期卡密规则：

- 系统随机生成。
- 默认 8 位大写字母数字。
- 兑换不区分大小写，用户输入小写自动转大写。
- 去除前后空格。
- 建议避免易混淆字符，如 `0/O`、`1/I`。
- 每张卡密只能兑换一次。
- 兑换成功后状态变为已使用。
- 记录兑换用户、兑换时间、授权专业/等级、到期时间。
- 卡密兑换在数据库层做原子操作，防止并发兑换同一张卡密。

卡密生成：

- 支持批量生成。
- 一次最多生成 100 张。
- 同一次生成使用相同专业、等级、授权时长、可选兑换截止日。
- 首期不做完整批次管理，但可保存简单 `generation_group_id` 方便筛选。

卡密列表：

- 首期不做 Excel 导出。
- 后台列表可查看完整卡密明文，便于运营手动复制分发。
- 可搜索卡密。
- 可按状态筛选：未使用（`unused`）、已使用（`used`）、已过期（`expired`，未使用且兑换截止日已过）。
- 注：授权到期是授权记录的状态，不是卡密的状态。卡密兑换成功后状态始终为"已使用"。
- 只有运营管理员和超级管理员可查看。

### 3.4 卡密购买引导

无授权用户登录后直接进入卡密兑换页。

页面包含：

- 卡密输入框。
- 兑换按钮。
- 已兑换授权列表。
- "还没有卡密？请联系销售/客服购买"提示。
- 联系电话、微信号、二维码图片、展示文案。

购买联系方式由运营后台配置：

- 联系电话。
- 联系微信号。
- 二维码图片。
- 展示文案。
- 是否启用。
- 修改记录审计日志。

## 4. 企业组织与企业授权

### 4.1 首期目标

首期企业侧只满足"平台运营集中维护企业组织、员工账号和授权"的模式。暂不开放企业后台。

企业客户优先考虑烟草企业，不考虑培训机构。培训机构作为未来扩展。

首期企业功能：

- 平台运营维护企业组织树。
- 平台运营创建或批量导入员工账号。
- 平台运营配置企业授权。
- 学员登录后基于企业授权判断可访问内容。

后续扩展：

- 地市管理员管理本级及下级县区。
- 县区管理员只管理本县区。
- 企业管理员导入员工、分配账号、查看数据。
- 企业报表和权限下放。

### 4.2 组织树与授权关系分离

企业行政层级和采购/授权关系必须分开建模。

企业组织树表示真实组织关系，例如：

```text
某省烟草公司
  └── 某地市烟草公司
        ├── A县区烟草公司
        └── B县区烟草公司
```

企业节点字段建议：

- 企业名称
- 企业层级
- 省/市/区县
- 上级企业
- 状态
- 联系人
- 备注

组织树约束：

- 系统禁止组织树循环引用，保存时校验。
- 首期组织树层级最多支持 4 级（省-地市-县区-基层）。

授权记录单独表示：

- 谁购买。
- 给谁使用。
- 授权哪些专业/等级。
- 授权多久。
- 授权额度多少。

必须区分：

- 组织归属：谁是谁的上级。
- 购买主体：谁付费。
- 使用主体：哪些企业/员工可使用。
- 授权范围：本企业及下级、指定企业列表。
- 有效期：授权开始、到期、续费。

企业授权记录字段建议：

- 授权名称/备注。
- 购买主体（企业节点）。
- 授权范围类型：本企业及下级 / 指定企业列表。
- 授权覆盖企业列表（指定模式下）。
- 专业。
- 等级。
- 账号额度。
- 已占用额度（系统自动计算）。
- 授权开始时间。
- 授权到期时间。
- 状态：有效 / 已过期 / 已取消。

### 4.3 企业组织停用

企业组织节点允许停用：

- 平台运营可停用企业节点。
- 停用后该企业及其员工不能继续使用企业授权内容。
- 数据、授权记录、员工账号、学习记录保留。
- 重新启用后，如授权仍有效，可恢复使用。
- 停用企业节点不等于删除企业。
- 停用操作记录审计日志。
- 企业节点停用时，如有该企业员工正在进行模拟考试或练习，系统自动保存当前作答并终止，记录状态为“已终止”（`terminated`）。用户重新进入时提示“企业授权已失效”。
- 企业节点停用时，即使员工同时拥有个人卡密授权覆盖同一专业/等级，正在进行的基于企业授权的考试/练习仍立即终止。终止后员工可基于个人授权重新开始新的练习或考试。

停用上级节点时，后台可选择：

- 只停用当前节点。
- 停用当前节点及全部下级。

启用时也应提供是否包含下级的选择。

### 4.4 企业授权范围

企业授权首期控制粒度：

- 专业
- 等级
- 有效期
- 账号额度

不控制到单套试卷或题库包。

授权范围支持两种：

1. 某企业及全部下级。
2. 指定企业列表。

适用场景：

- 地市公司统一采购，覆盖自己和全部县区。
- 县区公司先采购，仅覆盖本县区。
- 地市后续引入时，可以配置覆盖范围，避免与已有县区授权重叠。

续费与调整：

- 续费创建新授权记录，与旧记录有效期首尾衔接。旧记录到期后自动失效，新记录接续。
- 不修改已生效授权的有效期和覆盖范围。
- 需要调整额度或范围时，创建新授权记录并取消旧授权，或等旧授权到期后新记录生效。
- 取消企业授权时，受影响员工立即失去该授权对应的内容访问权。如有员工正在考试/练习，同 §4.3 企业停用处理。

指定企业列表中企业被停用：

- 被指定企业停用后，该企业及其员工不再享受这条授权，但授权记录本身不变。
- 重新启用企业后，如果授权仍有效且该企业仍在指定列表中，则恢复。
- 不做授权范围自动清理。

### 4.5 授权额度

企业授权账号额度按购买主体总额度共享。

规则：

- 一条企业授权有一个总账号额度。
- 授权覆盖范围内的员工共享该额度。
- 员工账号所属企业在授权覆盖范围内、员工账号启用、授权有效时，员工自动获得该专业/等级权限，并占用额度。
- 不做运营手动给员工分配专业/等级。
- 创建/启用员工时需要校验是否超过相关授权额度。
- 超过额度时运营后台提示异常，不允许继续启用或新增占用额度的员工。
- 额度校验在数据库层做原子操作，防止并发超额。

### 4.6 授权重叠

首期不允许同专业/等级的企业授权范围重叠。

规则：

- 企业授权创建或调整时，系统检查覆盖企业范围。
- 如果同一专业/等级、有效期重叠，并且覆盖到同一个企业节点，则不允许保存。
- 运营需要调整授权范围，避免重复覆盖。
- 这样同一员工在某专业/等级上最多匹配一条企业授权，额度占用清晰。
- 个人卡密授权不受该限制。个人授权和企业授权可以并存。

### 4.7 员工调动与离职

员工调动：

- 员工当前所属企业可以变更。
- 新作答记录使用新的企业归属。
- 历史学习记录保持作答时企业归属快照，不随账号当前企业变更。
- 后续企业报表按作答时企业快照统计。

额度处理：

- 员工变更所属企业时，系统自动释放原企业相关授权额度。
- 同时校验新企业相关授权额度是否充足。
- 如新企业对应专业/等级的授权额度已满，后台提示异常但仍允许调动，员工暂时无法享受该专业/等级的企业授权。
- 额度释放和占用在同一事务中原子执行。

员工离职：

- 运营可将员工账号从企业节点解绑。
- 解绑后不再继承企业授权。
- 不默认停用账号。
- 用户如有个人卡密授权，仍可作为个人用户登录。
- 如果没有个人授权，登录后进入卡密兑换页。
- 企业授权产生的历史记录保留企业归属快照。

员工解绑企业后：

- 员工本人不能查看以前通过企业授权产生的历史报告，除非重新获得对应专业/等级授权。
- 如果个人重新兑换同专业/等级卡密，则可恢复查看该专业/等级下自己的历史记录。

## 5. 授权访问统一规则

个人授权和企业授权取并集：

- 只要账号通过任一渠道获得某专业/等级有效授权，即可访问对应内容。
- 企业授权通过账号所属企业推导。
- 个人卡密授权直接绑定账号。

授权展示：

- 不合并隐藏多条授权。
- 我的授权列表分别显示：
  - 来源：个人卡密 / 企业授权。
  - 专业/等级。
  - 生效时间。
  - 到期时间。
  - 当前状态：有效 / 已过期 / 已取消。

授权取消：

- 个人卡密授权不可取消（与 §3.1 一致）。
- 企业授权可由超级管理员/运营管理员提前终止（取消）。
- 取消后状态变为“已取消”，受影响员工立即失去对应内容访问权。如有员工正在考试/练习，同 §4.3 企业停用处理。
- 取消操作记录审计日志。
- 已取消授权不可恢复，需创建新授权。

授权失效：

- 授权到期或取消后，前端隐藏对应专业/等级内容。
- 对应历史报告和学习记录在学员端隐藏。
- 后台数据保留。
- 重新获得对应专业/等级授权后恢复可见。

授权失效时进行中考试/练习处理：

- 所有授权失效场景（自然到期、主动取消、企业停用）均立即终止用户正在进行的对应专业/等级的练习和模拟考试。
- 如用户正在模拟考试中，系统自动保存当前作答并终止考试会话，记录状态为“已终止”（`terminated`）。
- 用户返回时提示“授权已失效，本次作答已终止”。
- 此规则同时适用于 §4.3 企业停用和 §5 授权取消场景。

站内到期提示：

- 授权到期前 15 天提示。
- 个人授权过期引导兑换新卡密。
- 企业授权过期提示联系所在企业或平台客服。
- 不做短信、微信、邮件提醒。
- 站内到期提示在用户每次进入首页时检查。同一条授权每天最多展示一次到期提醒，用户关闭后当天不再重复。提示位置：首页顶部通知条。

其他站内提示：

- 卡密兑换成功/失败需要即时提示。
- 兑换失败原因需清晰区分，如卡密不存在、已使用、已过兑换截止日。
- 模拟考试评分中、评分完成、评分失败通过模拟考试记录状态展示。
- 授权失效、企业解绑、企业停用导致内容不可访问时，学员端给出明确提示。

学习记录归属：

- 学习记录永远归属用户本人。
- 如果作答时该专业/等级存在有效企业授权，则同时记录企业归属快照。
- 如果只有个人卡密授权，则不记录企业归属。
- 如果同一专业/等级同时存在企业授权和个人授权，默认记录企业归属，因为企业授权覆盖了该学习权益。

后续企业报表权限预留：

- 地市管理员可查看本级及下级县区范围内明细和汇总。
- 县区管理员只看本县区明细和汇总。
- 普通企业角色只看汇总。
- 平台运营全局可查。

账号体系隔离：

- 后台用户账号与学员账号是两套独立体系（详见 06 §6）。
- 同一自然人如需同时使用学员端和后台端，需分别持有学员账号和后台账号。
- 两套账号的手机号可以不同，互不关联。
