## 1. 管理后台划分

管理后台首期只支持 PC Web，不做移动端适配。

后台分两个产品入口：

1. **内容后台**
   - 面向内容老师、出题人。
   - 管理题库题目、材料、试卷、组卷、发布/下架、知识点树维护。

2. **运营后台**
   - 面向平台运营人员。
   - 管理企业组织、企业授权、个人卡密、用户账号、资源文件、Markdown 知识库、AI/RAG 配置和日志。

两个后台共用账号体系和后端权限体系，但入口、菜单和使用体验分开。

2026-06-24 标准/高级版角色分离补充要求后台按工作区分离：

- `ops_admin` 登录后进入系统运营后台工作区，只能访问用户、企业、授权、`redeem_code`、资源、知识库、日志等运营能力。
- `content_admin` 登录后进入内容后台工作区，只能访问 `question`、`material`、`paper`、知识点树、内容 AI 草稿/评审等内容能力。
- `org_standard_admin` 和 `org_advanced_admin` 登录后进入组织后台工作区，权限必须限制在自身 `organization` 范围内。
- 每个后台工作区必须有角色感知的登录落点、可见退出入口、清晰导航和无权限状态。
- 后台直接访问无关工作区路由时必须返回清晰拒绝或无权限状态；隐藏菜单不是权限边界。

后台通用交互规则：

- 列表页默认分页，每页 20 条，支持调整为 50 / 100。
- 列表均支持按关键字段排序（点击表头切换升序/降序）。
- 筛选条件变化后自动刷新列表，不需要手动点击搜索。
- 批量操作（如批量生成卡密、批量导入员工）需二次确认弹窗。
- 危险操作（停用、下架、取消授权）使用红色按钮并要求二次确认。
- 操作成功/失败后显示 Toast 提示。
- 关键写操作（授权创建/调整、员工批量导入、卡密生成）使用乐观锁或数据库原子操作防止并发冲突。
- 并发冲突时提示“数据已被其他操作更新，请刷新后重试”。
- 后台 UI/UX 优化必须先完成设计产物，再进入实现。设计产物至少覆盖信息架构、导航、工作区分离、列表/详情布局、表单流程、空/错/加载状态、无权限状态、升级状态、冲突警告、确认弹窗、组件复用计划、目标路由和允许实现范围。

## 2. 审计日志

首期记录后台关键操作和用户关键行为。

后台关键操作：

- 发布试卷。
- 下架试卷。
- 生成卡密。
- 企业授权创建/调整/取消。
- 企业节点停用/启用（含级联选项记录）。
- 账号停用/启用。
- 员工创建/批量导入。
- 员工企业解绑。
- 重置密码。
- 购买联系方式修改。
- 模型配置修改。
- 知识点节点新增/编辑/排序/停用/移动/重命名。
- Markdown 发布。
- 向量重建触发。
- 资源停用/启用。
- 员工企业变更（调动）。
- 卡密兑换失败（记录失败原因、尝试兑换的手机号）。

用户关键行为：

- 卡密兑换。
- 模拟考试交卷。
- AI 评分失败重试。

审计日志字段建议：

- 操作人。
- 操作时间。
- 操作类型。
- 操作对象。
- 变更摘要。
- IP / 设备信息。

审计日志查看：

- 运营后台提供审计日志列表页。
- 支持按操作类型、操作人、时间范围筛选。
- 支持按关键字搜索操作对象。
- 日志只读，不允许修改或删除。
- 首期不做日志导出。
- 首期不设日志保留期限和归档策略，数据永久保留。后续数据量增长显著时再考虑归档方案。

## 3. AI 调用日志

AI 调用和 RAG 命中进入专门调用日志。

记录字段：

- 用户。
- 企业。
- 专业。
- 等级。
- AI 功能类型：主观题评分、AI 提示、AI 讲解、知识点推荐等。
- 模型供应商。
- 模型名称。
- 模型版本或配置标识。
- Prompt 版本。
- 输入 Token。
- 输出 Token。
- 调用耗时。
- 成功/失败状态。
- 估算成本。
- 关联对象：试卷、题目、作答记录。
- RAG 初召回数量。
- rerank 状态。
- 最终 Top 3 引用片段。
- 引用资料名称和章节路径。
- evidence_status。

AI 调用日志查看：

- 运营后台提供 AI 调用日志列表页。
- 支持按功能类型、用户、企业、专业、等级、成功/失败状态、时间范围筛选。
- 支持查看单条日志详情（含完整输入输出摘要）。
- 首期提供简易成本汇总：按日/月、按功能类型、按模型统计调用次数和估算成本。
- 日志只读，不允许修改或删除。

## 4. 运营后台功能清单

首期运营后台需要支持：

### 4.1 用户管理

- 用户列表：显示手机号、姓名、注册时间、账号状态、绑定企业、授权状态。
- 支持按手机号/姓名搜索。
- 支持按状态（正常/停用）、用户类型（个人/企业员工）筛选。
- 可查看用户详情：基本信息、授权列表、企业绑定信息。
- 可停用/启用用户账号。
- 可重置用户密码。
- 首期不做用户删除。

### 4.2 企业组织管理

- 企业组织树可视化展示（树形结构）。
- 支持新增、编辑、停用/启用企业节点。
- 停用时提供是否级联下级的选择。
- 支持在企业节点下查看/管理员工列表。
- 员工列表支持搜索、筛选、单个创建、批量导入、解绑。
- 员工批量导入必须提供可复用导入模板或等效模板下载/说明；模板只包含员工与 `organization` 绑定所需字段，不包含 `profession`、`level`、`edition` 或 `orgAuthScopePublicId`。
- 导入预览可展示预计继承的授权范围和额度影响；这些是计算结果，不是导入输入。
- 可查看企业节点的授权情况。

### 4.3 企业授权管理

- 企业授权列表：显示购买主体、授权范围、专业/等级、`edition`、额度/已占用、有效期、状态。
- 支持新建授权、取消授权。
- 新建授权时必须显式选择 `edition = standard | advanced`。
- 支持运营受控标准转高级升级入口，语义为创建或维护 `auth_upgrade.source_type = ops_manual`，不得覆写原始 `org_auth`。
- 支持一个商业企业授权包覆盖多个 `profession + level` 组合；系统必须展开为原子授权范围用于冲突、额度、有效期、取消和审计。
- UI 必须在提交前展示授权包摘要、展开后的原子范围、冲突警告、额度影响和有效期差异。
- 新建时校验授权范围重叠（同专业/等级不允许覆盖到同一企业节点）。
- 支持查看授权详情：覆盖企业列表、员工占用明细。

### 4.4 卡密管理

- 卡密生成支持单张生成和指定数量生成（同批次同专业/等级/时长/截止日）。
- 生成前必须显式选择 `profession` 和 `level`。
- 卡密列表：支持按状态（未使用/已使用/已过期）筛选、按卡密号搜索。
- 可查看卡密详情：生成信息、兑换信息（如已使用）。
- 卡密值只允许在受控查看/复制流程中用于运营分发；普通列表、审计日志、验收证据和提交文档不得明文暴露 `redeem_code`。
- 首期不做卡密导出和卡密作废。

### 4.5 购买联系方式配置

- 配置联系电话、微信号、二维码图片、展示文案。
- 支持启用/停用。
- 修改记录审计日志。

### 4.6 资源与知识库管理

- 资源列表：显示资源名称、专业、等级、资料类型、状态、上传时间。
- 支持上传资源文件。
- 支持查看资源详情：原始文件下载、Markdown 预览、状态流转记录。
- Markdown 校对发布界面（详见 05 §6.6）。
- 手动触发向量重建。
- 支持停用/启用资源。

### 4.7 日志查看

- 审计日志列表（详见 §2）。
- AI 调用日志列表（详见 §3）。

### 4.8 超级管理员额外功能

- AI 模型供应商配置（新增、编辑、启用/停用）。
- 模型用途映射（评分、讲解、提示、知识点推荐分别指定模型）。
- 备用模型配置（客观题讲解/提示的 fallback 模型）。
- API Key 密文展示（只显示后 4 位），修改时完整输入。

## 5. 内容后台功能清单

首期内容后台需要支持：

### 5.1 题库管理

- 题目列表：显示题干摘要、题型、专业/等级/科目、状态、知识点、标签。
- 支持按专业、等级、科目、题型、状态、标签、知识点筛选。
- 支持按关键字搜索题干文本。
- 可新建题目（按题型引导表单）。
- 可编辑未锁定题目。
- 可停用题目（已锁定题目也可停用）。
- 可复制题目为新题。
- 可查看题目被引用的试卷列表。

### 5.2 材料管理

- 材料列表：显示材料标题、专业/等级/科目、状态。
- 支持搜索和筛选。
- 可新建、编辑（未锁定时）、停用、复制。
- 可查看材料被引用的题目和试卷列表。

### 5.3 试卷管理

- 试卷列表：显示试卷名称、专业/等级/科目、状态、试卷类型、年份、总分、题目数。
- 支持按试卷名称、专业、等级、科目、状态、试卷类型、年份筛选。默认按更新时间倒序。
- 可新建草稿试卷。
- 草稿试卷：组卷界面，从题库选题、设置分值、调整结构。
- 可发布试卷（触发强校验）。
- 可下架已发布试卷。
- 可复制试卷为新草稿（详见 02 §4.2）。
- 可绑定/下载试卷原始文件。
- 可查看试卷的模拟考试记录数量（只读统计）。

### 5.4 知识点树维护

- 知识点树按专业展示（树形结构）。
- 支持新增、编辑、排序、停用节点。
- 支持移动节点（修改父节点）和重命名。
- 可查看节点绑定的题目数量。
- AI 知识点推荐结果查看和修正（在题目编辑页内）。

### 5.5 内容 AI 草稿与评审

标准/高级版角色分离补充要求内容后台提供 MVP 可发现入口：

- `content_admin` 可从内容后台导航中进入 `AI出题` 和 `AI组卷`。
- 生成结果进入内容 AI 草稿/评审域，不得直接写入正式 `question` 或 `paper`。
- 正式采用必须经过内容人员 review、编辑、校验、来源/评审人归属和 `audit_log`。
- Provider、Prompt、原始 AI 输出、完整生成内容和成本校准仍受后续审批门禁约束；验收证据只记录入口、状态、允许/拒绝结果和脱敏摘要。

## 6. 后台角色与权限

首期后台角色：

| 角色       | 可访问后台  | 权限范围                                   |
| ---------- | ----------- | ------------------------------------------ |
| 超级管理员 | 运营 + 内容 | 全局权限，含 AI 模型配置                   |
| 运营管理员 | 运营        | 企业、授权、卡密、用户、资源、知识库、日志 |
| 内容老师   | 内容        | 题库、材料、试卷、知识点树                 |

规则：

- 一个用户可拥有多个角色。
- 内容老师首期不做专业/等级权限隔离，可维护全部专业/等级内容。
- 运营管理员不能修改 AI 模型配置。
- 后台账号与学员账号分开管理：后台用户由超级管理员创建，不使用学员注册流程。
- 后台用户列表由超级管理员维护：可新建、编辑角色、停用后台用户。
- 后台用户首期不做自助注册。
- `ops_admin` 不得访问内容作者工作区或创建内容草稿。
- `content_admin` 不得访问系统运营的全局用户、全局企业授权、全局 `redeem_code` 或 Provider/成本治理能力，除非后续任务另行批准。
- `org_standard_admin` 只能管理自身 `organization` 范围内员工并查看企业授权/状态，不得管理企业训练或 AI 生成。
- `org_advanced_admin` 可管理自身 `organization` 范围内员工、查看授权/状态、管理企业训练，并使用组织高级 AI 出题/组卷能力。

### 6.1 组织后台角色边界

组织后台是企业管理员的一等工作区，不等同于系统运营后台：

- `org_standard_admin` 入口：员工管理、组织授权/状态查看、基础组织信息查看。
- `org_advanced_admin` 入口：员工管理、组织授权/状态查看、企业训练管理、组织 AI 出题/AI 组卷。
- 组织后台所有列表、详情、统计和操作都必须按 `organization` 范围过滤。
- 组织后台不得暴露全局用户、全局 `org_auth`、全局 `redeem_code`、系统审计、Provider、成本校准、支付、staging/prod 或内容作者能力。
- 组织后台与系统运营后台可以复用组件、表单、服务和审计模式，但权限边界必须由路由、菜单、服务层和审计共同保证。

### 6.2 后台用户安全

后台用户会话管理：

- 登录成功后签发 Token，有效期 8 小时（后端配置）。
- 允许同一后台账号多设备登录（后台多人协作场景常见）。
- 连续登录失败 5 次后，账号锁定 15 分钟。
- 锁定只阻止新登录，不影响已有活跃会话。已登录用户可继续使用。
- 后台用户密码规则与学员端一致：至少 8 位，须包含字母和数字。

后台用户创建流程：

- 首个超级管理员通过系统初始化脚本或种子数据创建，不通过后台界面。
- 后续后台用户由超级管理员通过运营后台创建。
- 创建时手动设定初始密码。
- 首期不强制首次登录修改密码。
- 后台用户不使用学员注册流程。
