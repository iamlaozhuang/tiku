## AI Generation Reuse Assessment

Answer to the reuse question: yes, the current implementation has real reuse for AI出题 and AI组卷.

| Shared concern                       | Shared implementation                                                                                                                                                                     | Role-specific boundary                                                                 |
| ------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------- |
| Task identity and count semantics    | `ai-generation-task-spec-contract.ts` defines `ai_question_generation` and `ai_paper_generation`, shared labels, default counts, structured preview kind, and redaction category.         | Page-level copy and default UI state.                                                  |
| Provider execution and redaction     | `route-integrated-provider-execution-service.ts` centralizes Qwen metadata, request limits, visible-content safety, redacted summaries, grounding checks, and structured preview parsing. | Admin and personal bridges decide owner/workflow context.                              |
| Provider instruction contract        | `route-integrated-provider-instruction-service.ts` centralizes output contract expectations for question set and paper draft.                                                             | Scene label and draft instruction differ by content, organization, or learner context. |
| Admin content and organization flows | `admin-ai-generation-runtime-bridge-service.ts` reuses shared task type and structured preview options.                                                                                   | `workspace`, `ownerType`, and organization public context vary.                        |
| Personal and employee learner flows  | `personal-ai-generation-route-integrated-provider-execution-service.ts` reuses shared Provider execution contracts and task specs.                                                        | Owner/quota context is resolved from personal vs employee session context.             |
| UI preview behavior                  | Admin and student AI pages both render `question_set` and `paper_draft` structured previews with shared field semantics.                                                                  | Role-specific page state and visible copy.                                             |

Static conclusion: prior repeated failures were not caused by a total absence of code reuse. The recurring failures came
from cross-cutting acceptance boundaries: role-specific authorization, runtime RAG coverage, Provider gating, structured
preview parse semantics, historical evidence drift, and browser/runtime proof not being kept as a single protected
acceptance baseline.
