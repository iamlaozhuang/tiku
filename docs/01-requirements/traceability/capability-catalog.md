## Scope

This catalog is derived from `docs/01-requirements/traceability/unified-standard-advanced-source-index.md`.

It is an audit and traceability artifact only. It does not create a use case catalog, edition delta matrix, technical
matrix, code audit, implementation queue, schema, migration, provider work, env/secret work, e2e work, deployment, PR,
force-push, payment, external-service work, or Cost Calibration Gate execution.

## Field Model

| Field                    | Meaning                                                                                                                    |
| ------------------------ | -------------------------------------------------------------------------------------------------------------------------- |
| `capabilityId`           | Stable audit id for this catalog.                                                                                          |
| `capabilityName`         | Human-readable capability label.                                                                                           |
| `sourceIds`              | Frozen source index ids that justify the capability.                                                                       |
| `editionScope`           | `standard_mvp`, `advanced_edition`, `unified_standard_advanced`, `future_scope`, or `blocked_gate`.                        |
| `requirementStatus`      | Requirement classification for later audit tasks.                                                                          |
| `blockedGates`           | Gates that still block implementation or execution.                                                                        |
| `conflictRefs`           | `CFX-*` ids from the source index that must be carried forward.                                                            |
| `auditUseOnly`           | `true` means the row is only a governance/audit reference and cannot be implemented as a product capability from this row. |
| `implementationEligible` | Whether a future scoped task may consider implementation after all declared gates and approvals are satisfied.             |
| `notes`                  | Boundary summary; no raw secret, provider payload, raw response, database URL, row data, or private content.               |

`implementationEligible: future_scoped_task_required` is not implementation approval. It only means the requirement can
be considered later if a queued task grants exact `allowedFiles`, gates, evidence, and approval.

## Edition-Aware Authorization Supplement

`CAP-ADV-AUTH-CONTEXT` is the source capability for the edition-aware source-of-truth model. Later implementation must
preserve these capability boundaries:

- `CAP-STD-PERSONAL-AUTH` remains responsible for standard personal authorization, while advanced personal authorization
  adds explicit `edition` and upgrade state rather than inventing a separate card system.
- `CAP-STD-ORG-AUTH-OPS-MANAGED` remains platform-managed organization authorization, while advanced organization
  authorization adds direct advanced issuance and auditable manual upgrade state.
- `CAP-ADV-OPS-AUTH-QUOTA` consumes authorization contexts and quota ownership summaries; it does not own provider
  execution, payment, production quota defaults, or Cost Calibration.
- Service-layer capability checks must use `effectiveEdition` and must not rely on frontend-only controls.

## Catalog Use Rules

1. Later use case catalog and edition delta tasks must cite `capabilityId` and `sourceIds`.
2. `auditUseOnly: true` rows must not seed implementation tasks.
3. `implementationEligible: blocked_until_gate_approved` rows remain blocked until a future task records exact
   `allowedFiles`, approvals, and passing gates.
4. `future_non_goal` rows must not be normalized into standard MVP or advanced first-release requirements.
5. `conflictRefs` must be carried forward unresolved unless a later task is explicitly scoped to adjudicate them.
6. Historical and blocked-gate sources may support provenance and risk notes only; they do not mark runtime coverage.
