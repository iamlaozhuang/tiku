# Unified Standard Advanced Source Index

## Freeze Statement

This document freezes the authoritative input set for the unified standard edition MVP and advanced edition audit
campaign as of 2026-06-14.

The 2026-06-21 edition-aware authorization supplement adds source rows for the confirmed edition source-of-truth
decision, ADR, and acceptance matrix. The supplement remains documentation-only and does not authorize implementation
or high-risk gate execution.

This is a source index only. It does not create the capability catalog, use case catalog, edition delta matrix,
technical landing matrix, code audit, implementation queue, schema, migration, provider work, env/secret work, e2e,
deployment, PR, force-push, payment, external-service work, or Cost Calibration Gate execution.

## Source Kind Model

| sourceKind                | Meaning                                                                                       |
| ------------------------- | --------------------------------------------------------------------------------------------- |
| `authoritative_source`    | Normative source for requirements, acceptance, or governance boundaries.                      |
| `supporting_source`       | Derived reading surface, story projection, handoff, or acceptance helper.                     |
| `historical_audit_source` | Historical evidence, audit, coverage, finding, or closeout context.                           |
| `blocked_gate_source`     | Boundary evidence that records future approval gates and blocked high-risk work.              |
| `excluded_source`         | Explicitly excluded from this input freeze and cannot drive later catalog extraction.         |
| `conflict_pending_source` | Cross-source semantic tension that must be carried forward until a later task adjudicates it. |

Authority levels:

- `normative`: source may define requirement semantics.
- `acceptance`: source may define acceptance granularity.
- `governance`: source may define process, scope, and blocked gates.
- `derived`: source helps navigation but does not override its upstream source.
- `historical`: source may be cited for prior findings, coverage, or context only.
- `blocked_gate`: source records a gate or approval package and cannot authorize execution by itself.
- `excluded`: source must not be used for this source freeze.
- `pending_decision`: source identifies unresolved or edition-scoped tension for later adjudication.

## Source Use Rules

1. Later capability and use case catalog tasks must cite `sourceId` values from this index.
2. Standard edition semantics are defined by `STD-REQ-*`; `STD-STORY-*` provides acceptance granularity.
3. Advanced edition semantics are defined by `ADV-SPEC-*` and `ADV-PLAN-02`; `ADV-DER-*`, `ADV-MOD-*`, and
   `ADV-STORY-*` are derived/supporting and cannot override upstream specs.
4. Historical audit sources may support finding provenance, coverage gaps, and risk notes only. They cannot directly
   drive implementation, rewrite requirements, or mark a capability complete.
5. Blocked gate sources identify future approval requirements. They do not approve provider, env/secret,
   staging/prod/cloud/deploy, payment, external-service, schema/migration, package/lockfile, e2e, PR, force-push,
   code audit, code fix, or Cost Calibration work.
6. Excluded sources must not be used during input freeze. Future tasks must explicitly expand their read/write scope if
   they need any excluded source.
7. Conflict-pending sources must be carried into later catalogs as unresolved or edition-scoped boundaries, not silently
   normalized.

The frozen source index is sufficient to start the next serial task, `unified-standard-advanced-capability-catalog`,
only after a future fresh task instruction. This task intentionally stops before catalog construction.
