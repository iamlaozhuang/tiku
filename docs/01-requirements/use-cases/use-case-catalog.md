# Unified Standard Advanced Use Case Catalog

## Scope

This catalog is derived from:

- `docs/01-requirements/traceability/unified-standard-advanced-source-index.md`
- `docs/01-requirements/traceability/capability-catalog.md`

It is an audit and traceability artifact only. It does not create a technical landing matrix, code audit,
implementation queue, schema, migration, provider work, env/secret work, e2e work, deployment, PR, force-push, payment,
external-service work, or Cost Calibration Gate execution.

## Field Model

| Field                    | Meaning                                                                                                              |
| ------------------------ | -------------------------------------------------------------------------------------------------------------------- |
| `useCaseId`              | Stable audit id for this catalog.                                                                                    |
| `useCaseName`            | Human-readable user or governance outcome.                                                                           |
| `capabilityIds`          | Capability catalog ids that justify the use case.                                                                    |
| `sourceIds`              | Frozen source index ids that justify the use case.                                                                   |
| `editionScope`           | `standard_mvp`, `advanced_edition`, `unified_standard_advanced`, `future_scope`, or `blocked_gate`.                  |
| `actor`                  | Primary actor or governance owner for the outcome.                                                                   |
| `outcome`                | Observable requirement-level result, without implementation design.                                                  |
| `requirementStatus`      | Requirement classification carried from the capability catalog.                                                      |
| `blockedGates`           | Gates that still block implementation or execution.                                                                  |
| `conflictRefs`           | `CFX-*` ids from the source index that must be carried forward.                                                      |
| `auditUseOnly`           | `true` means the row is only a governance/audit reference and cannot seed implementation from this catalog.          |
| `implementationEligible` | Whether a future scoped task may consider implementation after all declared gates and approvals are satisfied.       |
| `notes`                  | Boundary summary; no raw secret, provider payload, raw response, database URL, row data, prompt, or private content. |

`implementationEligible: future_scoped_task_required` is not implementation approval. It only means the use case can be
considered later if a queued task grants exact `allowedFiles`, gates, evidence, and approval.

## Catalog Use Rules

1. Later technical matrix or code audit tasks must cite `useCaseId`, `capabilityId`, and `sourceId` values.
2. `auditUseOnly: true` rows must not seed implementation tasks.
3. `implementationEligible: blocked_until_gate_approved` rows remain blocked until a future task records exact
   `allowedFiles`, approvals, and passing gates.
4. `future_non_goal` and `future_non_goal_for_standard` rows must not be normalized into standard MVP or advanced
   first-release requirements.
5. `CFX-*` conflicts must be carried forward unresolved unless a later task is explicitly scoped to adjudicate them.
6. Historical and blocked-gate sources may support provenance and risk notes only; they do not mark runtime coverage.
