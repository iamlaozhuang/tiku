# 题库系统全栈品味十诫 (Code Taste Ten Commandments)

这是本项目的**最高技术底线**。无论是哪位 AI 智能体（Claude Code / Codex / Antigravity）介入本项目的开发，都必须严格遵守以下十条绝对戒律。

任何违反以下准则产出的代码，将被直接视为废品，拒绝合入。

## 前端与 UI 品味 (Frontend & UI Taste)

1. **禁止廉价视觉感**：禁止使用默认的 `Inter` 字体；禁止使用纯黑色（`#000000`）；禁止使用不合理的紫蓝色渐变（The Lila Ban）。所有颜色必须基于 Tailwind 的 Zinc/Slate 中性色体系与我们预定义的 Design Tokens。
2. **强制交互状态完备性**：禁止生硬的无状态切换。任何数据拉取和交互，**必须**显式实现 Loading（骨架屏/加载效果）、Empty（数据为空时的优雅空状态）和 Error（错误回退）状态。
3. **强制微动效物理反馈**：任何可点击元素（如按钮），必须带有非线性的弹簧动画反馈（Spring Physics）或下压缩放反馈（如 `active:scale-[0.98]`），禁止毫无阻尼感的生硬反馈。
4. **Tailwind 类名绝对秩序**：所有组件和页面的 Tailwind 类名**必须**符合官方逻辑排序规则（由 Prettier 插件在提交时强行拦截格式化）。

## 后端与数据库品味 (Backend & DB Taste)

5. **拒绝 N+1 查询风暴**：在 Drizzle ORM 中查询关联数据时，**必须**使用 Relational Queries (`db.query.*.findMany({ with: {...} })`)，严禁在 `for` 循环中发起到数据库的 `select` 查询。
6. **强制强类型 Schema 驱动**：严禁在业务层手写（拼接） SQL 字符串；禁止使用会造成数据丢失的强制推送命令，所有库表变更必须走正规的 `drizzle-kit generate` 与 `migrate` 流程。
7. **标准化 API 响应契约**：所有的服务端 API，无论成功还是失败，必须始终返回标准格式的数据包：`{ "code": number, "message": string, "data": any }`，严禁随意调整外层结构或直接抛出堆栈。

## 业务逻辑与代码整洁 (Logic & Clean Code)

8. **消灭垃圾注释**：禁止写“说明代码在干嘛”的废话注释（如 `// 将 a 的值赋给 b`）。注释**只能且必须**用于解释业务上下文或特殊决策（如 `// 采用指数退避算法防止高峰期接口限流`）。
9. **有意义的命名法则**：禁止使用 `data`, `res`, `temp`, `x`, `info` 等毫无意义的变量名。所有函数名必须是明确的“动词-名词”结构（如 `fetchMarketData`），且严格遵循单复数约定。
10. **坚持数据不可变性 (Immutability)**：禁止直接修改状态对象或数组的内部属性（如 `arr.push()` ），任何状态更新**必须**使用扩展运算符（Spread Operator）或 `map/filter` 等纯函数返回新对象。

---

> **AI 交付要求**：在每次功能交付或代码修改完成后，智能体必须基于以上十诫输出一份简短的 Checklist 合规自检报告，未经核对单校验，严禁声称任务完成。
