# Issue Tracking

## Purpose

This directory records reverse-flow work: bugs, failed validation, blocked tasks, residual technical debt, and human decisions required to unblock automation.

## Rules

- Do not delete issue records after a fix. Add resolution notes instead.
- Failure reports must cite the task id, command, observed output, suspected cause, and required decision.
- Use this directory only for defects, blockers, and risk records. Positive execution evidence belongs in `docs/05-execution-logs/evidence/`.

## Subdirectories

- `bug-reports/`: task failures, validation failures, and reproducible defects.
