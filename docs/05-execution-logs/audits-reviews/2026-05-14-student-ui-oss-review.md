## 1. 答题端基础结构模板 (UI Skeleton)

基于 `docs/01-requirements/modules/03-student-experience.md` 规范，学员端需优先保障移动端体验（H5/Web），涵盖倒计时、答题卡、切题交互等功能。

### 候选一：Flexy Quiz / Xeven Quiz (作为组件参考)
- **技术栈**：React / TypeScript / Tailwind CSS
- **开源协议**：MIT
- **核心优势**：
  - 高度现代化的视觉设计，原生的响应式（Mobile-first）布局方案。
  - 内置了倒计时、分数统计面板、状态选项等基础的答题交互逻辑代码。
- **局限性**：这些大多是 Demo 级的项目，无法支撑复杂的数据流传和“错题本”复杂的页面状态。
- **借鉴建议**：不全盘引入，但可以剥离其中的“题目卡片平滑切换动画”、“答题卡（Question Grid）”和“倒计时圆环组件”等核心 UI 代码。

### 候选二：Vant UI (React 版) / Ant Design Mobile
- **技术栈**：React + TS
- **核心优势**：工业级的移动端组件库。对于后续向微信小程序方向靠拢或保持移动端原生体感非常有帮助。
- **借鉴建议**：在项目中作为基座组件库引入，以保障滑动、弹窗等在低端手机浏览器上的性能表现。

## 2. 核心渲染器 (富文本与公式)

烟草行业的考试（例如技能鉴定类）可能涉及一定量的表格、代码块以及公式渲染。我们需要绝对安全且高性能的解析方案。

### 推荐方案组合：`react-markdown` + `rehype-katex`
- **方案组成**：
  1. `react-markdown`：目前 React 生态中最正统的 Markdown 渲染引擎（基于 AST 而非 `dangerouslySetInnerHTML`，防范 XSS 安全风险极其有效）。
  2. `remark-math`：解析 Markdown 语法中的 `$` 和 `$$` 数学符号标记。
  3. `rehype-katex`：使用 KaTeX 引擎将解析出来的数学语法转为 HTML，相比 MathJax 速度更快，极其适合移动端（H5）加载。
- **集成优势**：
  在题目组件中，直接将接口返回的包含公式的 Markdown 文本丢入此组件，它便能安全且高速地完成图文和公式混排。
- **备选辅助库**：`Prose UI`，一个基于 Tailwind 的排版优化库，能让渲染出来的 Markdown 拥有非常棒的学术/极客排版风格。

## 3. 答题端状态管理最佳实践 (防作弊与防丢失)

通过查阅 Github 上优质的 Online Exam 项目的源码，我们提取了以下关于“状态保护”的最佳架构：
- **纯客户端防丢失**：利用 `zustand` 结合 `persist` 中间件，将考生的答题快照以毫秒级防抖（Debounce）存入 LocalStorage。
- **防切屏检测**：监听移动端的 `visibilitychange` 事件和 `blur` 事件。
- **服务端同步机制**：不采用每次点击实时 HTTP 请求，而是每隔 15 秒或切换大题时发送一次批量心跳（Heartbeat API）同步答案，既省服务器资源，又能保证体验。

## 🟣 补充调研：长尾与垂直领域替代方案 (第 5 轮)

### 跨端脚手架与国内生态适配 (Taro vs Uni-app)
根据 Tiku 需求中提及的“多端策略”以及未来可能向微信小程序迁移的考虑，在纯 Web 方案之外，我们补充调研了跨端脚手架的开源答题模板。

- **Uni-app (Vue 生态)**：国内极度繁荣。在 Github 或 DCloud 插件市场上有大量的开源/半开源的答题模板（例如 `uni-exam`）。其优势在于一键发行到微信、支付宝、H5 和 App，且生态内有很多现成的“答题卡抽屉”、“左右滑屏切题”的组件。
- **Taro (React 生态)**：由京东开源，允许我们继续保持 React + TS 的技术栈。相比 Uni-app，针对“考试系统”的现成轮子较少，一般推荐直接 Clone 类似 `taro-react-template` 等优质脚手架，然后手写状态机。
- **结论与推荐**：由于项目组硬性要求 React 体系，如果确定必须上微信小程序，则**强烈建议**答题端摒弃纯 Next.js 的 H5 方案，改用 **Taro (React)** 构建。我们可以将第 3 轮评选出的 `react-markdown` 逻辑封装在 Taro 的自定义组件中，实现一份代码编译出 H5 与小程序的双端一致体验。
## 🟢 补充调研：基于极简全栈架构的开箱即用脚手架 (第 7 轮)

针对学员端的富文本展示，如果我们采用纯 Web/H5 方案（暂缓开发原生小程序），我们为您寻找到了一款基于 Markdown 的极佳的开源组件底座：

### 推荐代码库：`bonartm/quizdown-js`
- **链接**：[Github](https://github.com/bonartm/quizdown-js)
- **核心优势**：这个库完美解决了“如何在移动端渲染包含数学公式的高难度题目”的问题。它允许完全使用 Markdown 来编写考试题，并且原生内嵌了 KaTeX 支持和触摸屏响应。
- **引入建议**：它不是一个完整的脚手架，而是一个极品的**前端组件引擎**。当我们在编写 Next.js 的学员答题页面时，可以直接引入这个组件处理服务器发来的 JSON/Markdown 题目流，极大缩减我们在移动端排版和适配数学公式的时间。

*(注：如果必须兼容小程序，前文提到的 `taro-react-template` 仍然是唯一的高分选项。)*
