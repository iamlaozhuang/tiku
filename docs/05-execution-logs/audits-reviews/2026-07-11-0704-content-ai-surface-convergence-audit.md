# 0704 内容 AI 页面收敛对抗式审查

## 结论

通过。修改限定在内容 AI 页面信息架构、用户文案和历史分页复用；生成请求、知识点范围、结果评审、正式采用、组织训练草稿和服务端权限逻辑保持不变。

## 审查矩阵

| 风险面     | 对抗场景                                                      | 结论                                                                                      |
| ---------- | ------------------------------------------------------------- | ----------------------------------------------------------------------------------------- |
| 角色边界   | 非内容角色进入内容 AI；内容角色访问组织训练数据               | 既有角色守卫未改变；内容与组织请求路径继续分离，前端导航不作为授权依据。                  |
| 版本边界   | 标准版组织通过页面重排获得 AI 入口                            | `standard-unavailable` 与 effectiveEdition 判定未改变；标准版仍不可用。                   |
| 题源边界   | AI组卷把生成正文或企业训练题当作平台正式题源                  | 文案和测试固定“方案生成后从平台正式题库本地选题”；既有正式题源装配合同未改变。            |
| 正式内容   | AI 结果跳过评审直接发布题目或试卷                             | AI出题只创建待审题目草稿；AI组卷只创建待审试卷；人工评审、驳回和发布校验链路保持。        |
| 数据边界   | 历史区展示 raw prompt、raw output、Provider payload 或内部 ID | 页面只显示脱敏业务摘要、状态和安全详情入口；敏感字段不进入 DOM 或 evidence。              |
| 状态完整性 | loading、empty、error、disabled 时误触发生成或翻页            | 生成按钮继续使用既有禁用判定；结果与历史状态分区；历史为空时共享分页显示 0 条且不可越界。 |
| 采用边界   | UI 重排改变评审采用、驳回或组织训练草稿写操作                 | 处理函数、请求路径和请求体未改；54 项定向测试覆盖既有内容与组织工作区行为。               |
| Provider   | 新文案或主操作隐式启用 Provider                               | 未改 Provider 配置与执行路径，未执行 Provider-enabled 行为；Cost Calibration 继续阻断。   |

## 品味复核

- 使用既有 design tokens、Lucide 图标和共享后台分页组件，无硬编码颜色或新依赖。
- 后台页面保持紧凑、可扫读；唯一主操作与生成条件相邻，结果和历史职责分离。
- 不建立万能页面框架，不复制分页实现，不改变 API 标准响应或 camelCase 合同。
- 标题层级、导航名称、按钮名称、空态和禁用态具有可访问语义。

## 残余风险

- 本任务执行边界禁止新增浏览器截图，因此未做新的像素级运行时验收；视觉判断基于既有私有截图和组件测试。
- 历史记录继续按当前页面的生成类型读取；本任务不新增服务端状态或时间范围筛选合同，避免用当前页客户端过滤制造伪筛选。
