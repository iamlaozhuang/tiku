# Phase 9 Requirements Runtime Gap Inventory

## Scope And Method

Task id: `phase-9-requirements-runtime-gap-inventory`

Branch: `codex/phase-9-requirements-runtime-gap-inventory`

Base: `master` at `3f9437b merge: phase 9 planning and queue seeding`

This inventory maps `epic-01` through `epic-06` P0/P1/P2 stories to current runtime, UI, API, and test evidence. It does not modify `src/**`, package files, schema, migrations, environment files, provider configuration, or production resources.

Status values:

- `covered`: current local runtime/UI/test evidence appears sufficient for the story-level acceptance target.
- `partial`: some route, service, UI, or test evidence exists, but at least one AC remains unproven or intentionally unavailable.
- `missing`: no meaningful runtime/UI/API evidence exists for the story-level acceptance target.
- `deferred_candidate`: non-P0 scope that may be deferred only with product approval in Phase 9 closeout.

In-scope clients and surfaces:

- Student mobile-first Web.
- Admin PC Web.
- REST API boundary under `/api/v1/` for future multi-client delivery.

Out of scope for this phase:

- WeChat mini program UI/client implementation.
- Real SMS, email, payment, production credentials, or real AI provider connections.
- External URLs or DTOs exposing auto-increment database `id`.

## Current Implementation Evidence Summary

- REST routes exist under `src/app/api/v1/**` for sessions, users, authorizations, personal auths, org auths, redeem codes, organizations, employees, questions, materials, papers, paper assets, student papers, practices, mock exams, exam reports, mistake books, model configs, resources, knowledge nodes, audit logs, and AI call logs.
- Student Web pages exist for `/login`, `/home`, `/profile`, `/redeem-code`, `/practice`, `/mock-exam`, `/exam-report`, and `/mistake-book`.
- Admin Web pages exist for `/ops/users`, `/ops/organizations`, `/ops/redeem-codes`, `/ops/ai-audit-logs`, `/content/questions`, `/content/materials`, `/content/papers`, and `/content/knowledge-nodes`.
- Unit tests cover current Phase 7/8 runtime slices and UI baselines, including student login/home/profile/redeem/mistake book, admin organization/org auth/redeem UI/runtime, student flow runtime, audit log runtime, and mock AI log runtime.
- E2E coverage is currently `e2e/local-business-flow.spec.ts` and `e2e/home.spec.ts`; Phase 8 evidence confirms Playwright browser coverage for the local product surface baseline, not full MVP acceptance.
- Remaining `createUnavailable...` boundaries exist for broad question/material/paper CRUD, paper assets, selected student workflow methods, model config mutations, knowledge/resource operations, reset password, broad admin user/org/auth operations, and selected AI/RAG paths.
