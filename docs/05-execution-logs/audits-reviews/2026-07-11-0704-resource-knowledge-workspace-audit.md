# 0704 资源与知识点工作区对抗式审查

## 结论

通过。修复限定在资源列表查询/展示、只读详情和知识点树交互；资源发布、索引重建、知识点创建/编辑/移动/停用仍使用既有服务端权限与业务校验。

## 审查矩阵

| 风险面     | 对抗场景                                                | 结论                                                                                 |
| ---------- | ------------------------------------------------------- | ------------------------------------------------------------------------------------ |
| 权限边界   | 非内容后台角色直接请求列表或写操作                      | 服务端角色守卫未改变；前端显隐不作为授权依据。                                       |
| 数据边界   | 通过列表或详情读取对象存储路径、向量、内容分片或内部 ID | 列表 DTO 保持裁剪；详情只显示可读内容和安全摘要；测试固定敏感字段不进入 DOM。        |
| 分页边界   | 数据超过 100 条、切换页大小、筛选后页码越界             | 查询由服务端分页；筛选和页大小回到第一页；本地目录与数据库来源的页偏移经过合并校正。 |
| 状态边界   | 草稿、已发布、待索引、可检索、索引失败、停用混用操作    | 查看、校对、发布、索引重建、启停独立呈现；服务端继续最终裁决状态迁移。               |
| 知识点目标 | 未选择节点即编辑、移动或停用                            | 三类写操作均禁用；选择状态由 `aria-selected` 和详情面板共同确认。                    |
| 知识点路径 | 移入自身或后代、手工输入父级标识                        | 父级改为可读选项并排除自身/后代；请求仍提交既有 publicId 合同并接受服务端校验。      |
| 版本边界   | 标准版/高级版或组织上下文被 UI 重排绕过                 | 本任务不改变 edition、authorization 或组织范围判定，也不增加组织数据请求。           |
| 敏感证据   | 测试、evidence 或日志记录资料全文、凭证或内部标识       | evidence 仅记录角色标签、路由标签、状态/问题类别、修复摘要和测试计数。               |

## 品味复核

- 使用既有 tokens、按钮、共享后台列表组件和 Lucide 图标，无硬编码颜色或新依赖。
- 后台布局保持紧凑、可扫读；资源列表使用表格，知识点使用主从树形布局。
- 不建立万能配置框架；资源查询扩展限于现有合同和仓储条件。
- API 响应封装、camelCase 字段和 publicId 路由约定保持不变。

## 残余风险

- 本任务执行边界禁止新增浏览器截图，因此未做新的像素级运行时验收；视觉判断基于既有私有截图和组件测试。
- 数据库资料若没有可读正文详情来源，详情仍提供安全元数据并明确显示“暂无可读内容预览”，不会伪造内容。
