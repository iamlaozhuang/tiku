# 2026-07-03 Source Landing 16 Package Role Acceptance Matrix

## Status

This is an acceptance execution preparation matrix only.

It does not execute acceptance, start a dev server, run browser validation, touch product source, connect to a database,
call a Provider, or claim release readiness, final Pass, production usability, Cost Calibration, staging/prod readiness,
or role-separated runtime Pass.

## Source Rule

Every row below cites at least one requirement source and at least one source/test/evidence anchor. Requirement point
status values are:

- `landed`: source landing evidence records a bounded source/test implementation.
- `partial`: source landing evidence records a bounded implementation plus a named residual gap.
- `follow_up_task`: requirement is accepted but needs a separate task before acceptance can pass end to end.
- `conflict_pending`: unresolved requirement conflict; none found in this preparation package.
- `not_needed`: explicitly outside first-release scope.

`super_admin` is not a primary role axis. It is covered as a privileged overlay in content, operations, and system-admin
rows and in the approval pack.

## `super_admin` Privileged Coverage Overlay

`super_admin` must be included in acceptance scenarios as a privileged coverage item, not as the ninth primary role axis.

Coverage expectations:

- Content: may access content resource management and content AI draft/review paths where the content workspace permits
  privileged access.
- Operations: may view/copy plaintext `redeem_code` in the same eligible operations UI exception as `ops_admin`; may move
  organization nodes where that first-release restriction is `super_admin` only.
- System admin: owns backend role/account management for `ops_admin`, `content_admin`, organization admins, and
  platform-level admin permissions.
- Model/Prompt: manages `model_provider`/`model_config`, can run redacted model connection test, and may view full
  registered Prompt text in product UI where DTO and role allow it.

Non-bypass expectations:

- `super_admin` does not bypass service authorization, public-id routing, context selection, redaction, evidence safety,
  Prompt/Provider payload restrictions, or release/production gates.
- `super_admin` coverage does not authorize browser acceptance, DB access, Provider calls, or deployment in this prep
  task.

Anchors: ADR-007, `D06`, `D10`, `D11`, `D18`, evidence packages 01, 02, 06, 07, 09, and 16.
