## Acceptance Sequencing Rule

Later execution must run these tracks serially unless a task explicitly splits a read-only preflight. If any track fails or
blocks, execution stops and opens a repair/provisioning task. The next acceptance rerun restarts from the beginning of
the full-chain sequence unless the repair task explicitly proves a narrower safe restart point.

## Non-Claims

This matrix is not runtime acceptance. It prepares coverage only and does not claim final Pass, release readiness,
production usability, Provider readiness, or DB readiness.
