# 2026-06-21 Edition-Aware Authorization Schema Migration Packet

## Goal

Implement the edition-aware authorization source-of-truth schema boundary from ADR-007 with a generated Drizzle
migration, without applying any local database migration.

## Fresh Approval

Current user fresh approval authorizes this packet only after startup checks confirm a clean `master` baseline. The
approved scope is limited to:

- `src/db/schema/auth.ts`
- related schema/unit tests, including `src/db/schema/auth.test.ts`, `src/db/schema/system.test.ts`, and
  `src/server/models/auth.test.ts`
- Drizzle migration files generated by `npx.cmd drizzle-kit generate`
- `docs/04-agent-system/state/project-state.yaml`
- `docs/04-agent-system/state/task-queue.yaml`
- `docs/05-execution-logs/task-plans/**`
- `docs/05-execution-logs/evidence/**`
- `docs/05-execution-logs/audits-reviews/**`

The approval allows generation of migration files, but does not allow applying local DB migrations, destructive DB,
staging/prod DB changes, provider/model calls, env/secret reads, dependency changes, payment, deploy, PR, force-push,
or Cost Calibration Gate execution.

## Implementation Plan

1. Materialize this fresh approval in task queue/state/evidence/audit.
2. Re-run `schemaMigration` capability gate and stop if it fails.
3. Write failing schema tests for:
   - `personal_auth.edition` and `org_auth.edition`;
   - `redeem_code.redeem_code_type`;
   - `auth_upgrade` table, source columns, status fields, and indexes.
4. Run focused schema/unit tests and record RED failure.
5. Implement minimal Drizzle schema changes in `src/db/schema/auth.ts`.
6. Run focused schema/unit tests and record GREEN.
7. Run `npx.cmd drizzle-kit generate`; inspect generated migration scope for unrelated drift.
8. Run `npm.cmd run lint`, `npm.cmd run typecheck`, `git diff --check`, hardening, closeout readiness, and pre-push
   readiness.
9. Commit, fast-forward merge to `master`, push `origin/master`, and clean the merged short branch before packet 2.

## Rollback And Stop Conditions

- If `drizzle-kit generate` requires applying a DB migration, stop and request fresh approval.
- If generated migration contains unrelated drift outside edition-aware authorization schema, stop before commit.
- If any validation requires reading or outputting a database URL, secret, token, Authorization header, provider
  payload, raw prompt, raw generated AI content, raw DB row, plaintext `redeem_code`, full `paper`, full `material`, or
  raw employee answer text, stop.
- Rollback before apply is a source revert plus removal of the generated migration and snapshot files from this branch.
- No local, staging, or production DB migration apply is approved in this packet.
