## Forced Branch Flow

每个 planned 分支执行同一流程：

1. 从最新 `origin/master` 开 `codex/` 短分支。
2. 强制阅读矩阵行、SSOT、相关代码和前置 evidence。
3. 写本分支 task plan，明确只解决当前矩阵行。
4. 先补 targeted test，再做最小源码实现。
5. 对抗式复核：权限、edition、空态、错误态、禁用态、数据红线。
6. 跑验证命令，写脱敏 evidence/audit。
7. 提交、快进合入 master、master 门禁、推送、删除短分支、确认 clean 且远端对齐。
